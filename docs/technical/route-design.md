# 路由设计

> 路由规划需考虑 App 和 Web 两端。App 端移动样式将作为 Web 端非查询类工单页面的复用参考。
> 详见 `cross-end-work-order-ui-strategy.md`。
> 本文档为早期路由规划参考。
> 正式 App 端路由以 Vue 工程实际路由为准。
> 以下路径仅表示页面信息架构，不作为正式路由强约束。


## 当前阶段

本阶段只定义路由结构，不开发具体业务页面。

## 路由规划

### 底部 Tab 路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/create` | 发起工单页 | 展示各工单类型入口卡片 |
| `/my` | 我的页 | 搜索、Tab、筛选、工单列表 |
| `/team` | 组内页 | 未开通，仅视觉占位 |

### 产品申请工单路由，信息架构参考

| 路径建议 | 页面 | 说明 |
|---|---|---|
| `/product-application/create` | 产品申请工单发起页 | 填写产品申请与预算核销信息 |
| `/product-application/detail/:id` | 产品申请工单详情页 | 查看工单、预算核销、功能订单、订单生成结果 |
| `/product-application/approval/:id` | 产品申请工单处理页 | 当前处理人通过 / 驳回 |

### 已有工单路由（预留）
