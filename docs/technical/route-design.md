# 路由设计

> 路由规划需考虑 App 和 Web 两端。App 端移动样式将作为 Web 端非查询类工单页面的复用参考。
> 详见 `cross-end-work-order-ui-strategy.md`。

## 当前阶段

本阶段只定义路由结构，不开发具体业务页面。

## 路由规划

### 底部 Tab 路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/create` | 发起工单页 | 展示各工单类型入口卡片 |
| `/my` | 我的页 | 搜索、Tab、筛选、工单列表 |
| `/team` | 组内页 | 未开通，仅视觉占位 |

### 预算申请工单路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/budget/create` | 预算申请表单 | 填写预算申请信息 |
| `/budget/detail/:id` | 预算申请详情 | 查看预算申请详情 |
| `/budget/approval/:id` | 预算申请审批 | 审批预算申请 |

### 已有工单路由（预留）

| 路径 | 页面 | 说明 |
|------|------|------|
| `/logistics/*` | 物流工单相关 | 已有功能 |
| `/service/*` | 客服工单相关 | 已有功能 |

## 路由配置位置

路由定义在 `src/app/router/` 目录下。

## 路由守卫

- 本阶段不实现认证和权限控制。
- 后续阶段按需补充。
