# 跨端工单 UI 复用策略

## 业务背景

本系统工单存在 **App 端**和 **Web 端**两端：

| 端 | 定位 | 样式特征 |
|----|------|----------|
| **App 端** | 移动端App / H5 体验，正式开发按 Vue 框架体系推进 | 底部 Tab 导航、卡片式布局、触摸优先 |
| **Web 端** | 桌面端管理后台 | 查询表格 + 移动端工单样式复用 |

### 关键决策

**App 端移动样式将作为 Web 端非查询类工单页面的复用参考。**

即：Web 端"发起工单、待办、已办、抄送我的、已发起"等页面，基本直接套用移动端工单样式和交互，不做独立的 Web 风格设计。

## 当前业务口径更新

历史“预算申请工单”口径已调整为“产品申请工单 / 产品申请预算核销”。

跨端 UI 复用策略仍可保留：
- 工单查询类页面使用管理后台样式；
- 发起、待办、已办、已发起、详情、审批等流程页沿用移动端工单样式。

Web 端正式开发技术栈已调整为 Vue2；
App 端正式开发按 Vue 框架体系推进，具体版本以正式 App 工程为准；
当前 React/Vite App 仓仅作为移动端样式和结构参考。


## 两端差异点

### App 端特有

| 元素 | 说明 |
|------|------|
| 底部 Tab 栏 | 发起工单 / 我的 / 组内（未开通） |
| 顶部 Header | 固定高度 48px，主色背景 |
| 手势交互 | 左滑返回、下拉刷新 |
| 安全区域 | 适配刘海屏、底部 home 条 |
| 页面最大宽度 | 430px 居中 |

### Web 端特有

| 元素 | 说明 |
|------|------|
| 工单查询页面 | 使用 Web 管理后台查询**表格样式**，不走移动端卡片 |
| 侧边导航 | Web 管理后台自有导航，替代底部 Tab |
| 面包屑 | Web 端页面层级导航 |
| 窗口自适应 | 内容区随浏览器宽度自适应 |

### 两端共用

以下页面/组件在两端共用同一套移动端视觉规范和交互结构：

- ✅ **发起工单页** — 入口卡片列表
- ✅ **待办页面** — 搜索框 + 状态 Tab + 工单卡片列表
- ✅ **已办页面** — 同上
- ✅ **抄送我的页面** — 同上
- ✅ **已发起页面** — 同上
- ✅ **工单详情页** — 详情展示 + 审批流程
- ✅ **审批操作** — 审批意见输入 + 按钮组
- ✅ **工单卡片组件** — 统一的卡片结构、状态色、排版
- ✅ **状态标签** — 统一的状态色对照
- ✅ **搜索框** — 圆角搜索输入框

## 复用策略

### 组件复用原则

当前阶段的“复用”主要指视觉规范、交互结构、组件拆分思路复用，不代表跨仓库直接复用 React 代码。

```
移动端组件（mobile.css + 组件库）
    │
    ├── App 端：按同样视觉和交互实现，+ 底部 Tab + Header
    │
    └── Web 端：非查询类页面复用，+ Web 布局壳子
                查询类页面独立设计（表格样式）
```

### 具体策略

#### 1. 样式文件复用

Web 端按与 App 端相同的类名体系和视觉规范进行复制/迁移，保持样式定义一致。

后续如有公共组件包再考虑代码级复用。当前阶段各自维护。

Web 端需要覆盖的布局差异：
- 移除 `.page-container` 的最大宽度限制
- 替换 `.app-header` 为 Web 端的面包屑或页面标题
- 替换 `.bottom-tab` 为 Web 端的侧边导航

#### 2. 组件级复用

Feature 模块中的组件按"纯展示组件"和"容器组件"分层：

| 层级 | 职责 | 复用方式 |
|------|------|----------|
| 展示组件（Presentational） | 纯 UI 渲染，接收 props | 结构和 Props 设计保持平台无关，便于 Web 端迁移或复刻 |
| 容器组件（Container） | 数据获取、状态管理、导航 | 各自独立实现或环境判断 |

#### 3. 布局适配

```
App 布局（LayoutApp）          Web 布局（LayoutWeb）
┌─────────────┐                ┌─────────────────────────┐
│   Header    │                │  Web 顶部导航栏          │
├─────────────┤                ├─────────────────────────┤
│             │                │  侧边栏 │               │
│  Content    │                │ 导航    │   Content     │
│  (430px max)│                │         │  (移动端样式)   │
│             │                │         │               │
├─────────────┤                │         │               │
│ Bottom Tab  │                │         │               │
└─────────────┘                └─────────────────────────┘
        │                              │
        └────── Content 共用 ──────────┘
```

### 本期影响

本阶段初始化时需要注意：

1. **组件设计**应避免硬编码 App 特有的布局元素（如 Header、Tab）。
2. **样式类名**使用通用命名（如 `.work-order-card` 而非 `.app-work-order-card`），便于 Web 端复用。
3. **Feature 目录**中 `components/` 下的组件应保持平台无关。
4. **App 特有布局**放在 `src/app/layout/` 下，不侵入 feature 组件。

## "组内"入口

**当前未开通，不纳入本期功能范围。**

- 组内功能在 App 端底部 Tab 中保留视觉占位。
- Web 端侧边导航中同样不开放"组内"入口。
- 待后续需求明确后再统一开发。
