# 会话启动卡

> 给新开的 AI 会话看的 `new-order-app` 速记版前情提要。

## 这个仓是什么

- App 参考与文档沉淀仓
- 当前主实现目录：`vue-reference/`
- 历史 React 归档目录：`legacy/react-reference/`

## 先记住这 5 件事

1. 这个仓不是当前正式 Web 主实现仓。
2. 当前最重要的代码入口是 `vue-reference/`。
3. `legacy/react-reference/` 是历史归档，默认降权。
4. 涉及 App 到 Web 的承接一致性时，必须同时对照 `../new-order-system`。
5. 先用 GitNexus 查结构和规则落点，再读长文档。

## 新会话建议阅读顺序

1. `AGENTS.md`
2. `CURRENT_TECH_STACK.md`
3. `CLAUDE.md`
4. `docs/ai/README.md`
5. `交付说明与承接清单.md`

## 本仓最常见任务

- 查 App 母版页面结构和交互
- 查 `vue-reference/` 的规则落点
- 对照 Web 是否漏承接、误承接
- 核对双仓之间的字段、状态、页面区块一致性

## 建议发给新会话 AI 的第一句话

```text
请先阅读 AGENTS.md 和 docs/ai/会话启动卡.md，再用 5 条以内总结你理解的当前仓库定位、主实现目录、历史归档目录、兄弟仓关系，以及本次任务该先看的目录，然后再开始处理任务。
```
