## 开发内容

### 1. 类型系统扩展（types/index.ts）

新增以下类型定义：
- `OrderAttempt.draftId`: 草稿链路ID，用于按草稿链路聚合
- `DraftLink`: 草稿链路类型（draftId + orderType + attempts + isDeleted + deleteReason）
- `CurrentResultDescription`: 5类固定枚举联合类型
- `ProductChangeSummary`: 产品变更摘要（changedSkuCount + totalQuantityDiff）
- `GroupResult.draftLinks`: 按 draftId 聚合的草稿链路数组
- `GroupResult.currentResultDescription`: 当前结果说明
- `GroupResult.currentMainReason`: 当前主原因
- `GroupResult.actualStoreCode/actualStoreName`: 实际专卖店变更信息
- `GroupResult.productChangeSummary`: 产品变更摘要

### 2. 订单结果区展示模型重构（DetailView.vue）

核心变更：
- 订单结果区主展示单元从"订单记录"切换为"专卖店订单申请结果单元"
- 每个专卖店对应一张结果卡片
- 卡片头显示专卖店编号+名称+当前结果说明标签
- 卡片内统一展示订单类型、订单号、当前主原因
- 订单创建历史从订单记录级提升到卡片级统一入口
- 历史区按 `draftId` 聚合展示草稿链路
- 草稿链路支持二级展开：链路头 → 关键事件

### 3. 当前结果说明 5 类枚举

实现自动判断逻辑（也可通过 mock 预置值精确控制）：

| 枚举 | 适用条件 | 标签样式 |
|------|----------|----------|
| 草稿提交失败 | 最新状态为草稿，提交失败 | 红色 |
| 审核驳回 | 最新状态为草稿，有审核驳回历史 | 红色 |
| 草稿已删除 | 草稿已被删除，无有效订单 | 红色 |
| 订单已创建 | 最新状态为已创建/客服已审核/财务已审核 | 绿色 |
| 订单已创建（信息变更） | 订单已创建，但专卖店或产品信息有变更 | 橙色 |

### 4. 当前主原因展示规则

每个卡片首屏只展示一条主原因：
- 草稿提交失败 → 显示失败原因
- 审核驳回 → 显示驳回说明
- 草稿已删除 → 显示删除理由
- 订单已创建（信息变更）→ 显示专卖店变更说明 + 产品变更摘要
- 订单已创建 → 不展示主原因

### 5. 订单信息变更承接

- 专卖店变更：`实际专卖店已变更为：{storeCode} {storeName}`
- 产品变更：`有 X 款产品共 X 个数量与申请工单内容不一致`
- 同时存在时：先展示专卖店变更，再展示产品变更摘要

### 6. Mock 数据补强（mocks/index.ts）

- 场景8（completed-retry）5个GroupResult全部添加 `draftLinks` 新结构
- 场景9（completed-all-failed）3个GroupResult添加 `draftLinks` + `currentResultDescription` + `currentMainReason`
- 新增场景11（completed-product-change）：产品变更场景
- 新增场景12（completed-store-product-change）：专卖店+产品双重变更场景
- 所有 `OrderAttempt` 添加 `draftId` 字段
- 场景映射总表更新至12个场景
