## 3. 关键变更说明

### 3.1 T1 — 专卖店选择页按App现有页面1:1回归（§7）

**问题**：此前 `CR-20260703-001 §1` 虽已完成一轮视觉统一，但实现为卡片式列表+返回按钮导航，与App现有页面的行式列表+搜索栏风格存在偏差。

**改造内容**：
1. **顶部导航**：去掉返回按钮（`<`）和标题"选择专卖店"，改为搜索栏（左侧"门店信息"标签 + 搜索框 + 右侧"取消"按钮），与App截图1:1对齐
2. **搜索历史记录**：按 App 原页面现有交互承接，含标题"搜索历史记录"+清除图标、历史关键词列表、点击回填；当前 Vue 参考实现阶段采用 `sessionStorage` 做轻量持久化占位，仅用于还原 App 现有页面体验
3. **列表样式**：从卡片式改为行式布局（编号在左、名称在右、底部分隔线），与App截图一致
4. **空态**：搜索无结果时显示"暂无搜索结果"
5. **搜索交互**：未输入时展示历史记录，输入后切换为搜索结果

**参考**：用户提供的两张App截图（2026-06-26 10:17:21 / 10:17:50）

**补充说明（搜索历史记录能力边界）**：
- 搜索历史记录能力属于 App 原页面现有交互承接的一部分，并非本轮新增业务规则或新增产品功能点；
- 当前 Vue 参考实现中采用 `sessionStorage` 进行轻量持久化，仅用于还原 App 现有页面体验；
- 正式工程中需结合真实客户端能力或统一历史记录机制，评估是否继续前端缓存，或改由后端 / 容器能力承接；
- 本轮已删除未使用的 `removeHistoryItem` 函数，保留能力边界收敛，不发散为编辑历史、拖拽排序、收藏置顶等扩展交互。

### 3.6 T6 — uuid moderate 依赖安全遗留评估（§8）

**当前状态**：
- 当前遗留为 `uuid` 相关 moderate 告警
- 来源于 `exceljs@4.4.0` 的传递依赖，并非本项目的直接依赖
- 本轮已完成来源、依赖层级与修复路径评估

**影响判断**：
- 当前主要影响为依赖安全告警层面
- 在本轮 Vue 参考实现与当前创建链路范围内，未识别出直接业务利用入口
- 但该项仍应作为**已知依赖安全遗留**持续跟踪，不应视为"已消除"

**本轮不处理原因**：
- 若采用 `npm audit fix --force` 高风险强制修复方案，可能导致 `exceljs` 功能异常，影响当前已完成的 Excel 导入解析与动态加载能力
- 因此本轮采取"记录风险 + 保留后续跟踪"的处理方式

**后续建议**：
1. 在正式工程中评估 `exceljs` 替代方案（如 `SheetJS` 社区版、`PapaParse` 纯CSV方案）
2. 关注 `exceljs` 官方版本更新，待其升级 `uuid` 传递依赖后同步升级
3. 如维持 `exceljs`，建议在正式工程中专项评估 `uuid` 版本升级可行性并做回归测试
