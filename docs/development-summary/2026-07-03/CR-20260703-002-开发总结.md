4. **空态**：搜索无结果时显示"暂无搜索结果"
5. **搜索交互**：未输入时展示历史记录，输入后切换为搜索结果

### 3.6 T6 — uuid moderate 漏洞评估与风险记录（§8）

**当前状态**：
- 漏洞来源：`uuid@8.3.2`，为 `exceljs@4.4.0` 的传递依赖
- 告警级别：`moderate`（npm deprecated: `uuid@10 and below is no longer supported`）
- 业务直接依赖：否，`uuid` 非本项目的直接依赖

**影响评估**：
- `uuid` 在 `exceljs` 中用于生成内部唯一标识，不影响业务数据安全
- 该漏洞为版本弃用警告，非活跃安全漏洞（CVE未关联）
- `exceljs@4.4.0` 为当前最新稳定版，`uuid@8.3.2` 为其锁定的传递依赖

**不采用 `npm audit fix --force` 的原因**：
- `--force` 会强制升级传递依赖，可能导致 `exceljs` 功能异常
- 当前为参考实现阶段，非生产环境，moderate级别风险可接受
- 后续如需彻底处理，应专项评估 `exceljs` 版本升级或替换为其他Excel解析库

**后续建议**：
1. 在正式工程中评估 `exceljs` 替代方案（如 `SheetJS` 社区版、`PapaParse` 纯CSV方案）
2. 关注 `exceljs` 官方版本更新，待其升级 `uuid` 传递依赖后同步升级
3. 如维持 `exceljs`，建议在正式工程中锁定 `uuid` 至 >=11 版本并做回归测试
