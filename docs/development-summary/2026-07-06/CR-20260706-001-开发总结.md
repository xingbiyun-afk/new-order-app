# CR-20260706-001 开发总结：产品申请工单详情页结构与字段层级优化

## 1. 变更单号

`CR-20260706-001`

## 2. 变更摘要

本轮在 `CR-20260703-003` 详情页驳回态与重新发起链路优化基础上，进一步收口详情页结构与字段层级问题。核心目标是让详情页"先看状态和结论、再看基础背景、再看明细与过程、最后看结果与附件"的阅读路径稳定下来。

### 2.1 区块顺序重排（§4）

按变更清单 §4.2 调整详情页整体区块顺序：

| 旧顺序 | 新顺序 | 调整原因 |
|---|---|---|
| 驳回信息 → 已到期提示 → 基础信息 → 发起信息 → 预算信息 → 明细 → 附件 → 审批流 → 订单结果 | 顶部状态区 → 驳回信息区(条件性) → 当前审批进度(仅处理中) → 基础信息 → 发起信息 → 预算与申请信息 → 明细 → 附件 → 审批流/订单结果(状态驱动) → 操作区 | 减少首屏次级字段堆叠；状态相关区块前置 |

### 2.2 字段层级重整（§6）

将字段按"识别工单身份 / 为什么发起 / 本次申请与预算关系"三类重整：

- **基础信息区**：工单号、当前状态、申请人、申请组织、创建时间（"工单类型"字段移除——与申请类型重复）
- **发起信息区**：申请类型、申请理由（从原"预算信息"区移出，独立表达"为什么发起"）
- **预算与申请信息区**：预算号、预算归属组织、预算使用范围、预算总额(净额/弱化)、本次申请额度(净额/突出)
- 去除字段重复：原"基础信息区"中的"工单类型"与"申请类型"语义重复，本轮移除前者

### 2.3 状态驱动的首屏差异化（§5）

- **处理中**（§5.1）：当前审批进度卡片前置（橙色边框 + 待处理节点 + 待处理人），其他与之前一致
- **已驳回**（§5.2）：驳回信息 Banner 首屏醒目，无当前审批进度卡片，底部"重新发起"操作区固定
- **已结束**（§5.3）：无驳回/审批进度卡片，订单结果区前移（绿色左边框突出），审批流区后置

### 2.4 产品申请订单明细卡片头增强（§7）

卡片头在原有明细编号(胶囊)、产品数量基础上，**新增分组金额徽章**（青绿色 `#E0F7F6` 背景，与创建页胶囊样式保持一致），折叠态即可快速看到每笔明细的金额。

### 2.5 统一提示风格（§9）

新增三类统一样式提示组件（`tip-banner`）：

- `tip-error`：红色调，用于驳回 Banner、订单生成失败、不可继续基于原单重提
- `tip-warning`：橙色调，用于已到期不可重提提示
- `tip-info`：青绿色调，预留
- 提示类型之间颜色、层级和区块风格统一，业务语义可区分

### 2.6 附件区可读性优化（§10）

- 文件名添加 `title` 属性（hover 显示完整名称）
- 文件大小行新增类型后缀（如 `3072 KB · PDF`），层级更清楚
- 多附件场景排布不变（已有 `border-bottom` 分隔）

### 2.7 审批流/订单结果区块位置收敛（§8）

按变更清单 §8 收敛：
- **处理中/已驳回**：审批流区 → 订单结果区（顺序不变，已驳回时审批流区不再抢驳回信息区焦点）
- **已结束**：订单结果区 → 审批流区（订单结果区层级提升，绿色左边框突出）
- 区块位置固定，本轮不做大范围重构

### 2.8 排版优化与同类问题修复（用户反馈后补充）

#### 2.8.1 产品明细卡片头两行布局（§7 补充）

- **问题**：卡片头一行挤满（标题+编号+产品数+金额），数字膨胀后排版混乱
- **修复**：改为两行布局
  - 第 1 行：产品申请订单明细 + 编号胶囊 + 折叠箭头（右对齐）
  - 第 2 行：款数 + 件数 + 分组金额（左 + 右分布）
- **抗极端场景**：金额 10+ 位、产品数 200+、款数 999、件数 9999 均不挤压

#### 2.8.2 汇总行四列网格（§7 补充）

- **问题**：汇总行"共 X 笔...共 Y 款 / Z 件"一段长文字，数字膨胀后挤压换行
- **修复**：改为四列网格布局
  - 每列独立：笔订单 / 专卖店 / 款 / 件
  - 数字大字(18px) + 标签小字(12px)，形成"数据看板"效果
  - 每列 `min-width: 0` + `white-space: nowrap`，数字不换行

#### 2.8.3 驳回信息区 meta 行基线对齐（§9 补充）

- **问题**：驳回人/日期不在同一基线，日期偏上
- **修复**：`.tip-meta` 加 `align-items: center` + `line-height: 1.5`

#### 2.8.4 文案口径统一：款/件（§7 补充）

- **问题**："X 个产品"口径不清（SKU 种类数 vs 总件数）
- **修复**：
  - 款 = SKU 种类数（`products.length`）
  - 件 = 数量合计（`Σ products[].quantity`）
  - 文案改为"X 款 · Y 件"
- **Mock 数据补强**：新增第 4 笔明细（10 款、370 件、¥150,637.00）

#### 2.8.5 CreateView 同类问题修复

- 底部金额合计栏：一行 `space-between` 改为两行（金额合计 + 预算提示）
- 预算提示区：三段 `space-between` 改为 `flex-wrap: wrap`
- 折叠态专卖店名：加 `overflow: hidden` + `text-overflow: ellipsis`

## 4. 关键决策

1. **移除"工单类型"字段**：原基础信息区"工单类型: 产品申请"与发起信息区"申请类型"语义重复，前者固定为"产品申请"无信息量，本轮移除以减少视觉噪音
2. **不调整审批节点命名**：模板中保留"审批节点 N"的占位命名（本轮不新建详情页规则文档）
3. **当前审批进度卡片仅在处理中展示**：通过 `v-if="isProcessing && currentApprovalNode"` 严格控制，已驳回/已结束场景不展示此卡片
4. **分组金额徽章位置**：放在卡片头右侧折叠箭头之前（视觉上与"产品数量"徽章并列），既保持折叠态摘要完整，又不与折叠交互冲突
5. **统一提示类组件化**：将原 `.rejection-banner` `.expired-banner` `.fail-reason` 三个分散样式统一为 `.tip-banner` + `.tip-error/warning/info` 三种变体，便于后续扩展
