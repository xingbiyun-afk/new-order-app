## 变更背景

本 CR 是对详情页规则的收口性建档，不引入新功能，重点解决：
1. 详情页正式规则明细文档缺失，规则分散在多处

## 主要改动

### 1. 新增《产品申请工单详情页面规则明细》文档

- 文件路径：`docs/product/work-order-budget/产品申请工单详情页面规则明细.md`（v0.2）
- 14 个章节，覆盖：
  - 页面定位与参考状态说明
  - 三态（处理中/已驳回/已结束）完整区块结构
  - 各区块展示规则与字段口径
  - Mock 场景与数据结构规范
  - 边界条件与白功能项跟踪

### 2. Mock 数据修正

- **处理中场景**（`mockWorkOrderDetail` 默认场景）：移除 `groupResults: mockGroupResults`
- **规则落地**：处理中场景禁止出现 `groupResults`/`rejectionInfo`/`reapplyCondition`/`retryHistory`
- **已驳回场景**：必须包含 `rejectionInfo` + `reapplyCondition`
- **已结束场景**：必须包含正式订单结果（`groupResults`）

### 3. 详情页轻量优化

- **移除导航栏右上角状态字**：删除 `DetailView.vue` 中的 `.header-status` 元素及对应 CSS
- **状态统一由内容区传达**：基础信息区"当前状态"字段为唯一状态展示入口
- **预占订单展示边界收口**：仅作为流程信息，仅出现在审批流起始节点，仅展示编号
