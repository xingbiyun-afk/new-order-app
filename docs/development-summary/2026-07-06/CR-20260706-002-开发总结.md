## 1. 审批流区优化（变更清单 §4）

### 1.1 节点名称字段化

- `ApprovalNode` 接口新增可选字段 `nodeName?: string`
- 所有 mock 场景的审批节点补充 `nodeName`（如"一级审批""二级审批"）
- 模板优先展示 `nodeName`，兜底显示"审批节点 ${i}"

### 1.2 节点类型视觉区分

新增 `getNodeClass()` 方法，按节点类型和结果返回 CSS 类名：

| 节点类型 | CSS 类名 | 视觉效果 |
|---|---|---|
| 发起工单（start） | `node-start` | 标题青绿色高亮 |
| 已通过（result=通过） | `node-passed` | 正常显示 |
| 当前待处理（result=待处理） | `node-current` | 橙色背景高亮 + 待处理人加粗 |
| 已驳回（result=驳回） | `node-rejected` | 红色背景高亮 + 驳回意见左侧红色边框 |

### 1.3 审批意见贴近节点

- `.node-remark` 改为卡片样式（浅灰背景 + 圆角）
- 已驳回场景下，驳回节点的 `.node-remark` 加左侧红色边框，视觉上紧贴驳回节点

## 2. 订单结果区优化（变更清单 §5）

### 2.1 预占库存订单展示收口（§6）

**变更前**：审批流区和订单结果区均展示"功能订单：XXX（状态）"

涉及修改位置：

- 无需额外修改（CR-20260706-001 已完成此部分）

## 8. Mock 合理性修复（用户反馈后，commit `8db25b4`）

用户反馈 3 个 mock 数据合理性问题，全部修复：

### 8.1 多笔预占订单支持

- `ApprovalNode.functionOrderNo` (string) → `functionOrderNos` (string[])
- 发起节点展示所有预占订单编号（场景 1：4 笔预占订单，chip 胶囊平铺 wrap 换行）
- 最终审批通过节点不展示预占订单和产品申请订单

### 8.2 失败原因去重

- 删除 `RelatedOrder.remark` 字段（与 `GroupResult.failReason` 内容重复）
- `GroupResult.failReason` (string) → `failReasons` (string[])，数组去重
- UI 使用 `dedupFailReasons()` 工具函数去重后多个 tip-banner 展示

### 8.3 多次失败重试与业务放弃

- 新增 `OrderAttempt` 接口：`{ attemptTime, status, failReasons? }`
- `GroupResult` 新增 `retryHistory?: OrderAttempt[]` 字段
- UI：折叠收起 + 最终态展示（"订单创建历史 N 次"），展开后时间线展示每次尝试
- `expandedRetries` Set 管理展开状态，`toggleRetry`/`isRetryExpanded` 方法
- 新增场景 9（`completed-retry`）：3 次重试最终成功 + 2 次重试业务放弃

### 8.4 同步修复

- 已结束场景审批流模板补读 `nodeName`（修复"审批节点 1/2"回退问题）
- `idSceneMap` 新增 `'9': 'completed-retry'` 映射

## 4. 白功能项：`uuid` moderate 风险复核

**结论**（变更清单 §10 要求）：
1. 风险来源：`exceljs@4.4.0` 的传递依赖，非当前业务直接依赖
2. 不采用 `npm audit fix --force`：会破坏现有 Excel 导入能力
3. 影响范围：仅 `parseFileImport()` 中的 Excel 解析能力
4. 后续建议路径：
   - 关注 `exceljs` 上游修复
   - 评估 `uuid` 升级可行性
   - 考虑 SheetJS 社区版 / PapaParse 纯 CSV 方案

**交付载体**：本开发总结 §4、项目日报 2026.07.06
