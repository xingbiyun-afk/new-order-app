### 3.4 预算选择页金额格式统一千分位 + 两位小数

**修改内容**：新增 `fmtMoney` 格式化函数，所有金额字段统一为千分位 + 两位小数格式。

```typescript
function fmtMoney(v: number): string {
  return v.toLocaleString('zh-CN', { minimumFractionDigits: 2, maximumFractionDigits: 2 })
}
```

**修改前**：`¥50,000` / `¥12,000` / `¥8,000`

**修改后**：`¥50,000.00` / `¥12,000.00` / `¥8,000.00`

**验证结果**：✅ 预算选择页三个金额字段均显示两位小数。

### 3.6 切换预算时确认提示并清空分组

**文件**：`vue-reference/src/views/work-order-budget/CreateView.vue`

**修改内容**：
1. 新增确认弹窗组件（遮罩层 + 白底圆角弹窗 + 取消/确定按钮）
2. 切换预算前检查是否已填写分组或商品
3. 若已填写，弹出确认提示，文案为："切换预算后，当前已填写的专卖店申请分组和商品明细将被清空，是否继续？"
4. 用户点击"确定"后，清空现有分组和商品明细，再跳转预算选择页
5. 用户点击"取消"后，保留当前内容，不跳转

```typescript
function goBudgetSelect() {
  if (hasGroupOrProduct()) {
    openConfirm('切换预算后，当前已填写的专卖店申请分组和商品明细将被清空，是否继续？', () => {
      store.storeGroups = [createDefaultGroup()]
      router.push('/product-apply/budget-select')
    })
  } else {
    router.push('/product-apply/budget-select')
  }
}
```

**验证结果**：✅ 选择产品后点击预算号区域，弹出确认弹窗。点击取消后内容保留；点击确定后跳转预算选择页。

**问题**：申请理由行使用 `justify-content: space-between`，导致"申请理由"标签被挤到两行显示（"申请理" + "由"），内容右对齐混乱。

**文件**：`vue-reference/src/views/work-order-budget/CreateView.vue`

**修改内容**：

**修改前**："申请理"换行 + "由"，内容右对齐

**修改后**："申请理由"完整一行，内容左对齐换行排列

**验证结果**：✅ 预算选择后"申请理由"四个字完整显示在一行，理由内容左对齐排列整齐。
