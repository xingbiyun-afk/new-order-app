## 一、变更背景

CR-20260630-002 已完成切换预算确认提示 + 清空专卖店，CR-20260630-003 已完成我的页列表补强。但发起页在专卖店选择和预算切换方面仍存在以下待收尾问题：

1. 已选专卖店后无法再次点击进入搜索页改选（只能先清空再选）
2. 未选专卖店时的占位文案"请输入专卖店编号"口径偏技术
3. 切换预算后 `collapsedGroups` 未被纳入清理范围，可能导致新分组异常折叠

本次变更对以上 3 个问题进行收尾修正。

---

## 二、逐项修改与验证

---

### 3.1 已选专卖店补齐"重新选择"能力

**文件**：`vue-reference/src/views/work-order-budget/CreateView.vue`

**修改内容**：
- 去掉 `@click` 的条件判断 `!group.storeCode ? ... : undefined`
- 无论是否已选专卖店，点击字段区域均进入搜索页
- 已选状态下箭头 `›` 始终显示，暗示可点击
- 清空按钮保留，与重新选择能力共存

**验证结果**：
1. 选择专卖店"31692 赵晋安 宋晓华" ✅
2. 点击已选专卖店区域 → 成功进入搜索选择页 ✅
3. 选择"31441 赵晋杰" → 成功改选 ✅

**影响范围**：仅影响点击交互行为，无数据清理行为。`clearStore()` 调用保持不变。

---

### 3.2 清空专卖店时同步清空商品明细

**状态**：无需代码修改，当前实现已符合要求。

**当前行为**：`clearStore()` 函数中，清空 `storeCode` + `storeName` 后，同步重置 `products` 为单条默认空商品。

**验证结果**：✅ 选择专卖店 → 选择产品 → 点击清空 → 专卖店和产品均被清空

---

### 3.3 未选专卖店占位文案调整

**文件**：`vue-reference/src/views/work-order-budget/CreateView.vue`

**修改内容**：占位文案从"请输入专卖店编号"改为"请选择订单归属专卖店"

**修改前**：`请输入专卖店编号`

**修改后**：`请选择订单归属专卖店`

**验证结果**：✅ 新分组未选时，显示"请选择订单归属专卖店"

---

### 3.4 切换预算确认后同步清空折叠状态

**文件**：
- `vue-reference/src/views/work-order-budget/CreateView.vue`
- `vue-reference/src/stores/workOrder.ts`

**修改内容**：
1. `workOrderStore` 新增 `clearCollapsed()` 方法，将 `collapsedGroups` 重置为空 `Set`
2. `goBudgetSelect()` 确认回调中，在清空 `storeGroups` 后，调用 `store.clearCollapsed()`

```typescript
// workOrderStore.ts
function clearCollapsed() {
  collapsedGroups.value = new Set()
}

// CreateView.vue - goBudgetSelect
openConfirm('...', () => {
  store.storeGroups = [createDefaultGroup()]
  store.clearCollapsed()  // 新增
  router.push('/product-apply/budget-select')
})
```

**验证结果**：
1. 选择专卖店后折叠分组（▸） ✅
2. 点击预算号区域 → 弹出确认弹窗 ✅
3. 点击确定 → 清空并跳转预算选择页 ✅
4. 选择新预算 → 返回发起页 → 新分组展开（▾） ✅
