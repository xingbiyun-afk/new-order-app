## 本轮目标

截至 2026-07-07，详情页相关工作已连续完成多个变更单。但在订单结果区"多次生成尝试 / 最终成功 / 最终失败 / 业务放弃"相关场景中，仍存在以下需要继续统一的问题：

1. 重试历史的外层摘要与展开区边界还不够清晰
2. 最终结果已成功时，外层卡片仍可能继续显示失败提示
3. 多次失败后业务放弃场景的备注展示口径需要收口
4. 订单结果状态术语仍存在"已生成/生成成功"等历史表述混用
5. 订单结果区对系统真实链路的承接关系尚未在文档与 Mock 中完全体现
6. 重试历史展开区排版存在可读性问题

本轮不新增业务功能，不改动正式订单系统规则，而是对**详情页订单结果区在"重试历史 / 最终结果状态 / 草稿回退 / 业务放弃备注 / Mock 规则映射"方向上的展示承接与术语口径**进行统一收口。

## 实现内容

### 1. types/index.ts — OrderAttempt 状态类型更新
- `status` 从 `'已生成' | '生成失败'` 更新为 `'已创建' | '客服已审核' | '财务已审核' | '草稿'`
- `failReason` 注释扩展为"失败原因或备注（仅失败/草稿时有值）"

### 2. mocks/index.ts — Mock 数据全面修正

**A. 全局状态术语统一**：
- 所有 `orderStatus: '已生成'` → `'已创建'`（涉及场景1/5/6/7/8/11，共10处）
- 所有 `orderStatus: '生成失败'` → `'草稿'`（涉及场景5/8/9/10，共6处）
- `OrderAttempt.status` 同步更新
- `failReasons` 文案更新为含草稿语义的新表述

**B. 场景8（PA202407050001）— 多次尝试后最终已创建**：
- g2(31441 赵晋杰)：移除外层 `failReasons`（最终成功时外层不显示失败原因）
- `retryHistory` 状态更新为"草稿/已创建"
- `failReason` 文案更新为含草稿语义

**C. 场景9（PA202407060001）— 多次失败后业务放弃**：
- 所有 `orderStatus` 更新为"草稿"
- 外层 `failReasons` 移除，改为 `remark` 承载最新业务备注
- `retryHistory` 状态和文案同步更新

**D. 所有场景的 `retryHistory`**：
- `status` 从"已生成/生成失败"更新为"已创建/草稿"
- `failReason` 文案更新为含草稿语义（如"订单自动提交失败回退到草稿"）

### 3. DetailView.vue — 订单结果区展示逻辑修正

**A. 新增 `getLatestOrderStatus` 辅助函数**：
- 获取订单最新状态，判断是否为成功状态
- 用于外层卡片决定展示内容

**B. 状态颜色映射函数更新**：
- 新增"已创建/客服已审核/财务已审核/草稿"四种状态的颜色映射
- 保留"已生成/生成失败"兼容处理

**C. 外层卡片展示逻辑修正**：
- 最终成功（已创建/客服已审核/财务已审核）：外层不显示失败主提示
- 草稿状态 + 有 remark：外层显示"当前为草稿状态"提示，展示 remark
- 草稿状态 + 无 remark + 有 failReasons：外层只展示最后一条失败原因
- 历史失败原因全部放入折叠区

**D. 重试历史展开区排版优化**：
- 改为 flex-direction: column 纵向布局
- 时间/订单号+状态独立一行（`result-retry-row1`）
- 失败原因/备注独立换行，避免挤压在最右窄列
- CSS 样式更新：添加 border-top 分隔线

### 4. 产品申请工单详情页面规则明细.md — 规则文档更新（v0.2 → v0.3）

**新增内容**：
- 6.10.8 订单结果状态固定口径（4种状态 + 不再使用"已生成"）
- 6.10.9 订单结果链路承接规则（完整链路：审批通过→取消预占→生成草稿→自动提交→已创建/草稿回退）
- 6.10.10 重试历史展示规则（默认折叠、外层最新摘要、业务放弃最新备注）
- 6.10.11 重试历史排版建议（长文本分行展示）
- 错误示例 6-8

## 关键决策

| 决策点 | 结论 | 理由 |
|--------|------|------|
| "已生成"是否完全删除 | 从 Mock 和类型定义中删除，页面保留兼容 | 类型定义和 Mock 数据必须统一新口径，但页面保留兼容处理以防上游数据未完全切换 |
| 外层 failReasons 的展示策略 | 草稿+remark 时展示 remark；草稿+无remark 时展示最后一条 failReason | 确保外层始终只展示最新一条摘要 |
| 业务放弃场景的备注载体 | 使用 RelatedOrder.remark | remark 是订单级别的最新说明，适合承载业务放弃的最终备注 |
