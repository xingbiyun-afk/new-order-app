# CR-20260702-002 开发总结 — 产品申请工单发起页面样式与体验优化

## 一、本轮目标

在 `CR-20260702-001` 完成预算选择页优化后，对发起页进行一轮样式与体验优化，解决以下问题：

1. 顶部信息区块占用过多移动端可视空间
2. 无草稿状态下的页面进入与返回规则不明确
3. 滚动后难以感知预算剩余额度
4. 明细区操作层级不清晰
5. 收起态信息不足
6. 术语不统一
7. 附件必传感知不足
8. 提交按钮缺乏前置禁用能力

## 二、实现页面清单

| 页面/文件 | 变更内容 |
|-----------|----------|
| `vue-reference/src/views/work-order-budget/CreateView.vue` | 发起页主体全部 11 项优化 |

## 三、主要组件与结构说明

### 3.1 新增状态与计算属性

| 名称 | 类型 | 说明 |
|------|------|------|
| `isApplyInfoExpanded` | `ref<boolean>` | 申请信息区展开/收起状态 |
| `showBudgetBar` | `ref<boolean>` | 滚动态预算摘要条显示控制 |
| `canSubmit` | `computed<boolean>` | 提交按钮可用状态计算 |
| `isFromSelectionPage` | `ref<boolean>` | 是否从选择页返回（避免清空状态） |
| `handleScroll` | `function` | 滚动事件监听处理 |

### 3.2 核心交互逻辑

**无草稿状态进入规则**：
- 从非选择页进入（且无驳回重提参数、无预算ID参数）→ 清空为空白态
- 从选择页返回（budget-select/store-select/product-select）→ 保留当前状态
- 驳回后重提场景 → 保留原预算锁定状态

**滚动态预算摘要条**：
- 监听 `window.scrollY`，超过 280px 且已选预算时显示
- 固定于页面顶部，仅展示剩余额度

**提交按钮禁用态**：
- 未选预算 → 禁用，文案"请选择预算"
- 未上传附件 → 禁用，文案"请上传附件"
- 未填写产品明细 → 禁用，文案"请填写产品明细"
- 以上均满足 → 可用，文案"提交"

## 四、关键业务规则承接说明

### 4.1 空白进入与返回重进规则（变更清单 §4）

- 当前产品申请工单发起页**不支持草稿**
- 用户未提交成功前，中途退出后再次进入 → 按新开页面处理，不保留未提交内容
- 从预算选择页/专卖店选择页/产品选择页返回 → 不属于退出重进，正常回填

### 4.2 预算信息区压缩（变更清单 §5）

- 预算信息区内边距从 14px 压缩至 12px
- 行间距从 10px/8px 压缩至 8px/6px
- 归属组织和范围合并为一行展示
- 驳回重提提示文案精简

### 4.3 申请信息区展开收起（变更清单 §5.3）

- 默认收起态：仅显示申请人 + 申请类型摘要
- 点击标题区域展开/收起
- 未选择预算时显示"选择预算后展开"

### 4.4 滚动态轻量预算摘要条（变更清单 §6）

- 预算信息区滚出可视范围后显示
- 仅展示"剩余额度" + 金额数字
- 样式轻量，不抢占填写区域

### 4.5 金额合计区预算关系提示（变更清单 §7）

- 在金额合计下方增加轻量提示行
- 超预算时红色警告："⚠ 已超过预算剩余额度"
- 未超预算时绿色提示："还可使用 ¥xxx"

### 4.6 明细区操作层级优化（变更清单 §8）

- "添加产品申请订单"为主操作，保持原样式
- "批量导入产品"降级为辅助操作，增加分隔线和说明文字
- 导入按钮样式弱化（实线边框 → 浅色背景）
- 增加警示文案："导入后将覆盖当前已填写内容"

### 4.7 明细收起态增强（变更清单 §9）

- 收起态新增分组小计金额
- 布局优化：专卖店信息 + 产品数量 + 小计金额

### 4.8 产品入口术语统一（变更清单 §10）

- 产品选择入口标签从"产品编号"改为"产品"

### 4.9 附件必传感知增强（变更清单 §12）

- 附件标题增加红色必传标识 `*`
- 未上传时增加红色"（必传）"提示

### 4.10 提交按钮状态化（变更清单 §13）

- 当前页可直接判断的缺失项前置禁用
- 禁用态灰色背景 + 对应提示文案

### 4.11 数量提示优化（变更清单 §11）

- 超量时突出红色警告："⚠ 已超过可申请数量，请调整"
- 达上限时黄色提示："已达到可申请上限 xxx"

## 六、Bug 修复记录

### 6.1 选择页返回后信息未回填

**问题描述**：选择专卖店或产品后返回发起页，已选择的信息未被带出。

**根因分析**：CR-20260702-002 新增的无草稿状态逻辑中，`isFromSelectionPage` 仅检查了 `route.query.from` 参数，但专卖店/产品选择页返回时通过 `storeCode`/`productCode` 参数回填，未携带 `from` 参数，导致页面被误判为"新进入"而清空了状态。

**修复方案**：判断从选择页返回时，同时检查 URL 中是否存在 `storeCode` 或 `productCode` 参数，有这些参数也说明是从选择页返回的，不应清空状态。

**修复文件**：`vue-reference/src/views/work-order-budget/CreateView.vue`

## 七、验收整改记录（验收后收尾）

### 7.1 初始化分支预算误清空修复（必改）

**问题**：`onMounted` else 分支中保留条件反向清空了预算。

**整改**：保留场景（选择页返回 / 驳回重提 / budgetId 回填）中仅执行 `initStoreGroups()` + `selectBudget()`，移除 `store.selectedBudget = null` 反向清空逻辑。

### 7.2 术语统一（必改）

| 原文 | 整改后 | 位置 |
|------|--------|------|
| `请选择产品编号` | `请选择产品` | 提交校验 toast |
| `请选择预算号` | `请选择预算` | 提交校验 toast |
| `预算号已锁定` | `预算已锁定` | 驳回重提拦截 toast |
| `字段顺序：产品编号` | `字段顺序：产品编码（产品编号）` | 批量导入弹窗提示 |

页面表单字段标签"预算号"保留不变（字段名本身）。

### 7.3 emoji 替换为 SVG 图标（必改）

| 位置 | 原 emoji | 替换为 |
|------|----------|--------|
| 批量导入按钮 | 📋 | SVG 文件图标（`stroke="#22BDB8"`） |
| 附件区 PDF 提示 | 💡 | SVG 信息图标（`stroke="#FF9800"`） |
| 导入弹窗文件上传 | 📁 | SVG 文件夹图标（`stroke="#999"`） |

### 7.4 批量导入按钮视觉弱化过度修复（必改）

**问题**：验收整改第一轮将按钮颜色过度弱化至 `#999`/`#fafafa`，导致几乎不可见。

**整改**：恢复为品牌色 `#22BDB8` 虚线边框 + 白色背景 + 品牌色文字/图标，清晰可见又不抢主操作。

### 7.5 魔法值抽成常量（建议）

`window.scrollY > 280` → `window.scrollY > BUDGET_SCROLL_THRESHOLD`
