## 一、变更背景

当前产品申请工单发起页仅支持逐条手动添加产品，SKU 数量多时效率低。业务需要整张工单维度一次性批量导入，系统按专卖店编号自动拆分明细。

本次变更聚焦：粘贴导入入口、数据校验规则、页面拆分承接、Excel/CSV 文件导入补充、模板下载。

## 三、逐项修改与验证

### 3.1 粘贴导入（核心功能）

**入口**：发起页"产品申请订单明细"区域顶部，与"添加产品申请订单"并列。按钮文案"📋 批量导入产品"，附轻量提示说明。

**弹窗结构**：
- 字段顺序醒目区块（浅橙色背景，加粗 "产品编号，数量，专卖店编号"）
- 格式说明（支持逗号、空格、Tab 分隔）
- textarea 输入区（等宽字体，自动行高）
- 📁 文件上传区（"选择文件"按钮 + 文件名 + "下载模板"按钮）
- 底部操作：取消 / 识别检查 / 确认导入

**文本解析**（`parseImportText`）：
- 支持逗号、空格、制表符分隔
- 跳过空行
- 字段顺序固定：产品编号、数量、专卖店编号
- 格式错误（不足3列）→ 逐行错误提示

**数据校验**（`validateImportRows`）：
| 校验项 | 层级 | Mock 实现 |
|--------|------|-----------|
| 产品编号不允许为空 | 硬拦截 | ✅ |
| 产品在核销范围内 | 硬拦截 | 查 mockProductOptions |
| 数量为正整数 | 硬拦截 | isNaN / <=0 检查 |
| 专卖店编号不允许为空 | 硬拦截 | ✅ |
| 专卖店在可选范围 | 硬拦截 | 查 mockStores |
| 层级5/不允许下单 | 硬拦截 | 暂未实现（注释标记） |
| 数量超限 | 软警告 | 数量 > maxQuantity |

**硬拦截 = 全有或全无**：任一硬拦截错误 → 展示全量错误列表 → 禁止导入。

**超量软警告**：允许导入到页面，但提交时通过 `skuTotalMap` 拦截。

**验证结果**（playwright-cli 实测）：
1. 空页面直接导入 3条数据 → 按专卖店拆分为2个明细 ✅
2. 已有数据时弹出覆盖确认 → 取消不导入 ✅
3. 无效 SKU + 格式错误 → 全量错误展示，禁止导入 ✅
4. 超量数据 → 警告提示，仍可导入，提交时拦截 ✅

### 3.2 按专卖店自动拆分

**Store 方法**（`importWorkOrderData`）：
- 遍历校验通过的 ImportRow 数组
- 按专卖店编号 `Map<storeCode, ImportRow[]>`
- 查找对应专卖店名称 → 创建 StoreGroup
- 相同编号归入同组，不同编号拆分

**验证结果**：
- 3条数据（31692×2 + 31441×1）→ 页面生成 2 个产品申请订单明细 ✅

---

### 3.3 覆盖导入交互

**实现**：复用现有 `openConfirm` 机制。

**流程**：
1. 点击"批量导入产品"
2. 若 `storeGroups.length > 0` → 弹出确认弹窗："批量导入后，将覆盖当前已填写的产品申请订单明细和产品信息，是否继续？"
3. 用户确认 → 调用 `clearAllGroups()` → 打开导入弹窗
4. 无数据 → 直接打开导入弹窗

**验证结果**：
- 已有数据时弹出确认 ✅
- 取消后页面数据保持 ✅
- 确认后正确清空并打开导入窗 ✅

---

### 3.4 Excel/CSV 文件导入

**函数**：`parseFileImport(file: File)` → `Promise<{ rows: string[], error: string | null }>`

**解析策略**：
| 格式 | 解析方式 | 
|------|----------|
| `.csv` | 纯 JS 按行解析，逗号 split |
| `.xlsx` / `.xls` | `xlsx` 库读取 |
| 其他格式 | 返回错误提示 |

**格式约定**：
- 第 1 行 = 表头（跳过）
- 第 2 行起 = 数据
- 列顺序：产品编号、数量、专卖店编号
- Excel 读取第 1 个 sheet

**数据流**：文件 → parseFileImport → 文本行数组 → 填入 textarea → 识别检查 → 确认导入（完全复用粘贴导入链路）。

**验证结果**：
- CSV 文件导入 → 3条数据识别为2个店 → 导入成功 ✅
- Excel (.xlsx) 导入 → 同样数据 → 识别检查成功 ✅
- 不支持格式 / 仅表头 → 错误提示 ✅
- 选择文件后可手动编辑 textarea ✅

---

### 3.5 模板下载

**函数**：`downloadImportTemplate()` — 生成 CSV 模板并触发浏览器下载。

**模板内容**：
```
产品编号,数量,专卖店编号
SKU001,5,31692
SKU002,3,31441
```

**格式选择说明**：CSV（UTF-8 BOM 编码），在 Excel/WPS 中自动适配列宽、中文正确显示。`xlsx`（SheetJS）社区版不支持列宽和单元格样式写入。

**验证结果**：
- 点击"下载模板" → 浏览器下载 `批量导入模板.csv` ✅
- 文件内容正确：表头 + 2 行示例 ✅
- 页面 URL 保持不变（无 hash 跳转） ✅
