## 三、问题修复记录

### 问题 1：底部Tab遮挡提交按钮

**现象**：发起页底部固定提交按钮被底部Tab栏遮挡

**根因**：二级页面不应显示底部Tab，但Tab栏未做路由判断

**修复**：在 App.tsx 中增加 `TAB_ROOTS` 判断，只有一级页面（/create, /my, /team）才显示Tab

---

### 问题 2：预算/专卖店选择后数据未回填

**现象**：从预算选择页或专卖店选择页返回发起页后，选择的数据未显示

**根因**：使用 React Context 传递状态，页面切换后 Context Provider 重新挂载导致状态丢失

**修复**：放弃 Context 方案，改用 URL Search Params 传递 `budgetId`、`storeCode`、`storeName`、`groupId`、`productCode` 等参数，发起页通过 `useSearchParams` 读取并回填

---

### 问题 3：分组折叠后 groupId 不一致

**现象**：从专卖店选择页返回后，专卖店未正确回填到对应分组

**根因**：每次组件重新挂载时 `createDefaultGroup()` 生成新的 groupId，与 URL 中的 groupId 不匹配

**修复**：使用 `sessionStorage` 持久化 `storeGroups` 状态，确保页面重新挂载时 groupId 一致

---

### 问题 4：我的页面无法滚动

**现象**：mock 数据增多后，我的页面内容超出视口但无法滚动

**根因**：`.page-content` 缺少 `overflow-y: auto`

**修复**：在 `mobile.css` 的 `.page-content` 中添加 `overflow-y: auto` 和 `-webkit-overflow-scrolling: touch`

---

### 问题 5：全局 header 多余

**现象**：发起页（一级页面）顶部出现青绿色 "工单处理中心" header，与截图不符

**根因**：App.tsx 中对所有页面都渲染了全局 header

**修复**：增加 `showHeader` 判断，仅一级页面（/create, /my, /team）显示全局 header，二级页面各自有独立 header

---

### 问题 6：发起页/我的页缺少 "工单处理中心" 标题

**现象**：发起页和我的页没有显示 "工单处理中心" 黑色标题

**根因**：之前移除了全局 header 但未在页面内添加标题

**修复**：
- 一级页面恢复全局青绿色 header "工单处理中心"
- 发起页和我的页内的黑色标题 "工单处理中心" 保留（与 App 截图对比后的最终样式）

---

### 问题 7：我的页面搜索+Tab区域缺少白底

**现象**：搜索框+Tab+筛选栏区域背景为灰色，与 App 截图不符

**根因**：该区域未设置白色背景

**修复**：将该区域包裹在 `backgroundColor: '#fff'` 的 div 中

---

### 问题 8：底部 Tab 图标不匹配

**现象**：底部 Tab 使用三个圆点图标，与 App 截图不符

**根因**：TabItem 组件使用通用圆点

**修复**：为三个 Tab 分别实现 SVG 图标：发起工单(+号)、我的(勾选框)、组内(六边形)

## 六、设计决策说明

### 6.1 为什么使用 URL Search Params 而非 Context

发起页 → 选择页 → 发起页 的跳转过程中，React Context 在页面切换后会重新挂载导致状态丢失。URL Search Params 方案：
- 状态持久化在 URL 中，页面刷新不丢失
- 无需全局 Context Provider，组件间耦合低
- 浏览器前进/后退行为自然

### 6.2 为什么使用 sessionStorage 持久化 storeGroups

专卖店选择页返回后需要 groupId 匹配。sessionStorage 方案：
- 页面重新挂载时 groupId 保持一致
- 会话结束后自动清理
- 不影响其他页面状态

### 6.3 为什么键值对布局替代纯文本行

旧实现使用"申请人：张三"纯文本，新 App 截图使用 `申请人`（灰色）+ `张三`（黑色）的键值对布局。键值对布局：
- 信息层级更清晰
- 与 App 截图 1:1 还原
- 扩展性更好（新增字段不破坏布局）
