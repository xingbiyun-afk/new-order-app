## 3. 当前已知问题

### 3.1 详情页场景验收样本仍缺少统一总表
虽然当前已有多组 Mock 场景与详情页路由入口，例如：

- `processing`
- `rejected-freeze`
- `rejected-expired`
- `rejected-nonfreeze`
- `completed`
- `completed-full`
- `completed-merged`
- `completed-retry`
- `completed-all-failed`
- `completed-store-change`

但当前仓库内仍存在以下问题：

1. **场景编号、scene key、工单号、主状态、核心验证点之间缺少统一总表**
   - 后续验收仍需要人工记忆“哪个路由 id 看哪个问题”；
   - 文档、Mock、开发总结中偶发存在“场景编号叫法不统一”的情况。

2. **详情页规则主文档虽已写规则，但对“验收样本如何使用”承接不够集中**
   - 规则写了，但缺少“样本级入口索引”；
   - 测试、截图、复验、AI 自检仍要翻代码或自己猜场景映射。

3. **Mock 入口访问方式缺少统一说明**
   - 当前既支持 `/product-apply/detail/:id`
   - 也支持 `?scene=xxx`
   - 但缺少正式文档明确说明“怎么进场景、推荐用哪个入口、各入口对应什么”。

4. **每个场景要验证什么、不能出现什么错误，还未集中枚举**
   - 例如：
     - 处理中不能出现正式订单结果；
     - 最终已创建不能继续显示失败主提示；
     - 业务放弃只展示最新备注；
     - 预占库存订单只在发起节点出现；
   - 这些虽然散见于不同 CR 和规则段落，但还未形成一份可直接验收的样本检查表。

### 3.2 页面容器与滚动承接规则仍缺少正式收口
在 `CR-20260707-002` 实施与验收过程中，已暴露并修正以下问题：

1. flex column + 100vh 布局下，`.page-content` 才是真正滚动容器；
2. `router.scrollBehavior` 只影响 window 滚动，对 `.page-content` 无效；
3. 列表 → 详情、详情 → 列表、路由切换时如不显式复位 `.page-content`，会出现：
   - 继承上一个页面的滚动位置
   - 页面切换后左移 / 双重滚动条
   - 首屏位置不正确

这些问题目前虽然已在实现层有补丁，但还未正式写成“Vue 参考实现页面容器承接规则”，后续新页面或 AI 继续接手时，容易重复出现相同问题。

## 4. 本轮变更目标

本轮目标共 5 项：

1. **建立详情页场景验收样本总表**
   - 在正式规则文档中集中枚举：
     - 场景 id
     - scene key
     - 工单号
     - 主状态
     - 核心验证点
     - 不应出现的错误

2. **统一 Mock 入口访问说明**
   - 明确 `/detail/:id` 与 `?scene=xxx` 的使用方式、推荐方式及映射关系；
   - 降低后续测试、截图、复验与 AI 自检成本。

   - 使其继续承担详情页当前直接执行依据角色。

4. **正式收口 Vue 参考实现的页面容器与滚动承接规则**
   - 明确实际滚动容器；
   - 明确路由切换时滚动复位策略；
   - 明确哪些问题由 `scrollBehavior` 解决，哪些问题必须由页面容器主动处理。

5. **同步协作文档与入口说明文档**
   - 让产品目录入口、Vue 参考实现入口、协作指南等能统一说明：

4. 不允许同时保留两个主滚动容器造成双重滚动条；
5. 不允许因容器布局设置不一致导致页面切换左右偏移、首屏位置异常。
