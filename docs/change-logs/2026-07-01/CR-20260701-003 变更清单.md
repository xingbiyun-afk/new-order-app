# CR-20260701-003 变更清单

> **变更单号**：`CR-20260701-003`  
> **变更名称**：仓库主工程去歧义与历史 React 工程归档  
> **变更日期**：2026-07-01  
> **适用仓库**：`new-order-app`  
> **变更类型**：工程结构收口 / 协作基线去歧义 / 历史工程归档

---

## 一、变更背景

当前仓库业务与文档口径已经明确：

- 当前正式承接方向为 **Vue / Vue3**
- 当前产品申请工单参考实现主目录为：`vue-reference/`
- 根目录中保留的 React + TypeScript + Vite 内容仅为历史阶段参考

但当前仓库仍存在明显的双轨冲突：

1. **文档口径已统一为 Vue**
   - `README.md`
   - `CLAUDE.md`
   - `CURRENT_TECH_STACK.md`
   - `WORKFLOW_GUIDE.md`
   - `docs/ai/项目协作基线.md`
   均已明确当前主方向为 Vue / Vue3

2. **工程入口仍默认指向 React**
   - 根目录 `package.json` 依赖和脚本仍以 React 工程为主
   - 根目录 `vite.config.ts` 为 React 插件配置
   - 根目录 `index.html` 和 `src/` 仍为 React 主入口
   - 根目录 `tsconfig*.json` 仍为 React JSX 配置

3. **实际已出现 AI 被误导的情况**
   - 新对话中的 AI 直接依据根目录工程信号，将当前仓库误判为 React 项目
   - 说明最小去歧义方案虽已降低误判概率，但尚未从根因上消除工程层误导

因此，本次变更决定直接执行 B 方案：

- **将根目录 React 历史工程迁移到独立归档子目录**
- **将根目录默认运行 / 构建入口切换为 Vue**
- **使文档、目录结构、脚本入口和 AI 协作基线完全一致**

---

## 二、本次变更目标

本次变更目标如下：

1. 将仓库默认工程事实从 React 切换为 Vue；
2. 将历史 React 工程迁出根目录主路径，避免继续占据主入口；
3. 使根目录 `npm run dev` / `npm run build` 默认作用于 Vue 参考实现；
4. 保留历史 React 工程作为参考，但明确归档定位；
5. 使 README、CLAUDE、CURRENT_TECH_STACK、WORKFLOW_GUIDE、项目协作基线与目录结构、脚本行为保持一致；
6. 将 AI 因根目录工程信号而误判技术栈的概率降到接近 0%。

---

## 三、核心执行方案

### 3.1 总体方案
本次执行 **B 方案**，具体为：

- 将根目录现有 React 主工程迁移至：
  - `legacy/react-reference/`
- 根目录保留：
  - 文档层
  - Vue 主工程调度入口
  - 当前技术栈声明与协作规则
- `vue-reference/` 继续作为当前主实现目录
- 根目录 `package.json` 调整为**调度型 package**
- 根目录默认脚本切到 Vue

## 五、根目录 `package.json` 调整要求

### 5.1 调整目标
根目录 `package.json` 不再作为 React 主工程 package 使用，而改为：

- 仓库级说明入口
- Vue 主工程快捷脚本调度器
- AI/协作工具扫包时的第一层纠偏信号

---

### 5.2 根目录脚本调整要求
根目录 `package.json` 最终至少应提供以下脚本：

```json
{
  "scripts": {
    "dev": "cd vue-reference && npm run dev",
    "build": "cd vue-reference && npm run build",
    "preview": "cd vue-reference && npm run preview",
    "dev:vue": "cd vue-reference && npm run dev",
    "build:vue": "cd vue-reference && npm run build",
    "preview:vue": "cd vue-reference && npm run preview",
    "dev:react-legacy": "cd legacy/react-reference && npm run dev",
    "build:react-legacy": "cd legacy/react-reference && npm run build",
    "preview:react-legacy": "cd legacy/react-reference && npm run preview"
  }
}
```

---

### 5.3 根目录依赖调整要求
根目录 `package.json` 不再保留 React 主工程依赖作为默认依赖集合。

处理原则：

1. 根目录移除直接服务于历史 React 工程的依赖；
2. React 相关依赖应跟随 React 工程一起放入：
   - `legacy/react-reference/package.json`
3. 根目录若保留 package，仅保留最小必要信息：
   - `name`
   - `private`
   - `description`
   - `scripts`

> 本次目标不是保留“双主工程”，而是明确：
> - 根目录默认 = Vue
> - React = 历史归档参考
