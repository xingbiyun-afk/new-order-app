# CR-20260702-001 产品申请工单发起页预算选择页面样式与体验优化

## 一、变更背景

当前产品申请工单发起页中的“选择预算”页面，已具备基础预算卡片展示与预算选择能力，但在实际使用中仍存在以下问题：

1. 预算卡片的信息层级不合理，预算号占据了过强的视觉位置，而用户最关注的“当前还能申请多少”不够突出；
2. 当前卡片展示字段更偏向预算管理视角，缺少申请人决策所需的关键信息，如预算归属组织、预算使用范围等；
3. 预算状态与时效特征缺少面向申请场景的标签化提示，用户难以快速判断“最近用过哪条预算”“哪些预算临近到期”“哪些预算已进入冻结期”；
4. 零剩余额度预算、冻结期预算与普通可申请预算之间的可选性差异未被清晰表达；
5. 预算维护侧已有成熟规则，但在申请侧选择预算页面尚未形成贴合移动端使用习惯的展示、排序和交互规则。

基于以上问题，需要对产品申请工单发起页及关联预算选择链路进行一轮样式与体验优化，以提升申请人理解成本、选择效率与使用稳定性。

---

## 二、变更目标

本轮变更目标如下：

1. 优化预算卡片的信息层级，使“剩余额度（净额）”成为预算选择时的第一视觉重点；
2. 补充申请人真正关心的辅助字段，帮助其理解预算归属与使用范围；
3. 引入面向申请场景的预算标签机制，提升预算识别效率；
4. 明确预算可见、可选、排序与冻结期特例规则；
5. 为后续沉淀“选择预算页面规则明细文档”打下统一实现基础。

---

## 三、变更范围

本轮变更聚焦以下范围：

1. 产品申请工单发起页中的“选择预算”页面；
2. 发起页中已选预算信息的回显与摘要展示；
3. 驳回后重新发起场景下，与预算选择相关的交互限制与提示。

本轮不涉及：

- 预算管理后台规则重做；
- 预算搜索范围规则重做；
- 预算号可搜索字段扩展；
- 草稿业务能力新增；
- 正式 API 协议重定义。

## 四、页面展示与样式优化

### 4.1 预算卡片信息层级调整

预算卡片需调整为以“额度判断”为核心的展示方式。

#### 调整要求
1. **预算号弱化显示**
   - 缩小字号
   - 降低视觉权重
   - 不再作为卡片首要视觉焦点

2. **“可用金额”更名为“剩余额度（净额）”**
   - 作为卡片第一优先级信息
   - 需明显强化展示，便于用户第一眼识别

3. **去除字段**
   - 已用金额
   - 申请中金额

4. **新增字段**
   - 预算总额（净额）
   - 预算归属组织
   - 预算使用范围

### 4.2 展示字段说明

#### 预算总额（净额）
用于帮助申请人理解该预算整体规模，与“剩余额度（净额）”形成对照。

#### 预算归属组织
用于说明预算实际归属的组织主体，便于申请人理解该预算是：

- 属于本人当前组织；
- 还是属于总部 / 区域 / 其他组织，但开放给本人所在组织使用。

#### 预算使用范围
用于说明预算可被哪些组织或对象使用，帮助申请人理解自己为何可见、可用该预算。

### 4.3 移动端卡片展示压缩规则

由于移动端卡片空间有限，需补充以下展示约束：

1. 预算归属组织默认单行展示；
2. 预算使用范围采用摘要式展示；
3. 超长内容允许截断，不在本轮新增完整展开弹层能力；
4. 本轮以“可快速判断、不过度挤占卡面空间”为优先目标。

## 五、预算标签机制

### 5.1 标签目标

预算标签用于帮助申请人在预算列表中快速识别更适合优先选择的预算，不用于替代预算状态本身。

### 5.2 标签类型

本轮新增以下标签：

#### 1）最近申请
满足以下条件时显示：

- 在用户权限范围内可见；
- 为用户最近一次**提交成功**的产品申请工单所使用的预算；
- 当前剩余额度（净额）大于 0。

说明：
- 仅以“提交成功”作为判定依据；
- 当前工单体系未规划草稿状态，因此不考虑草稿口径；
- 若用户尚无成功提交记录，或该预算当前剩余额度为 0，则不显示该标签。

#### 2）15天内即将到期
满足以下条件时显示：

- 在用户权限范围内可见；
- 距离 `budgetExpiryDate` 小于等于 15 天；
- 当前剩余额度（净额）大于 0。

#### 3）本季度可申请
满足以下条件时显示：

- 在用户权限范围内可见；
- 已进入预算期间可申请时间范围；
- 当前剩余额度（净额）大于 0。

#### 4）申请冻结中
满足以下条件时显示：

- 在用户权限范围内可见；
- 已进入 `freezeStartDate`；
- 当前时间尚未达到 `budgetExpiryDate`；
- 当前剩余额度（净额）大于 0。

说明：
- 原“到期冻结中”标签名称调整为“申请冻结中”；
- 该标签用于表达“预算进入新增申请冻结期，但尚未正式到期”的状态。

### 5.3 标签展示规则

1. **同一卡片只显示 1 个标签**
2. 显示命中标签中的**最高优先级标签**
3. 标签优先级如下：

   1. 最近申请  
   2. 15天内即将到期  
   3. 本季度可申请  
   4. 申请冻结中

## 六、预算可见与可选规则

### 6.1 不展示的预算范围

以下预算不在选择预算页面展示：

1. 草稿
2. 已作废
3. 已到期

### 6.2 展示但弱化的预算范围

以下预算可继续展示，但需弱化处理并排在后面：

1. 用户权限范围内可见；
2. 剩余额度（净额）为 0。

### 6.3 零剩余额度预算可选性

明确规则如下：

- 剩余额度（净额）为 0 的预算**不可选**
- 前端应以明显不可选态展示
- 如用户点击，应给予明确提示，例如：
  - 当前预算剩余额度不足，无法用于申请

### 6.4 权限范围约束

预算列表仅展示**用户权限范围内可见**的预算，页面不得突破现有权限规则。

---

## 七、排序规则

预算列表排序规则调整如下：

### 7.1 一级排序：标签优先级
按以下优先级排序：

1. 最近申请
2. 15天内即将到期
3. 本季度可申请
4. 申请冻结中

### 7.2 二级排序：距离到期日
在标签优先级基础上，再按照距离 `budgetExpiryDate` 的时间升序排序，即：

- 距离到期日越近，排序越靠前

### 7.3 零剩余额度预算排序
在上述规则范围内，**剩余额度（净额）为 0 的预算整体后置**。

### 7.4 兜底排序
若标签优先级与距离到期日均相同，建议按以下顺序继续兜底：

1. 剩余额度（净额）降序
2. 预算号升序

## 八、搜索规则补充

本轮延续既有结论：

- 预算选择页仅支持**预算号搜索**

并补充以下说明：

1. 搜索结果仍需遵循本轮预算可见规则；
2. 搜索结果仍需遵循预算不可选规则；
3. 搜索命中后，不打破本轮已明确的状态过滤与排序逻辑。

## 九、卡片交互与反馈优化

### 9.1 卡片点击范围
预算卡片整体应支持点击选择，不应仅依赖小范围点击区域。

### 9.2 选中态
预算卡片需提供清晰的选中态表现，包括但不限于：

- 边框高亮
- 背景轻微变化
- 明确的已选标识

### 9.3 不可选预算反馈
对于不可选预算，点击后应提供清晰反馈，不得出现“看似可点但无反应”的情况。

其中：

- 零剩余额度预算：提示剩余额度不足
- 申请冻结中预算（普通新发起场景）：提示当前预算已进入申请冻结期，不可用于新发起

### 9.4 空态区分
需区分以下两类空态：

1. **当前无可申请预算**
   - 适用于按权限与状态过滤后无预算可展示的情况

2. **搜索无结果**
   - 适用于用户按预算号搜索后未命中任何结果

两类空态文案应避免混用。

## 十、冻结期规则与驳回后重提特例

### 10.1 普通新发起限制

根据预算维护侧既有规则：

- `freezeStartDate` 默认取 `endDate + 45 天`
- 自 `freezeStartDate` 起，不可再发起新的产品申请工单（用于预算核销）

因此在选择预算页面中：

- 进入申请冻结期的预算，不可用于普通新发起场景的预算选择

### 10.2 冻结期驳回后重提特例

若某张已使用该预算号的产品申请工单，在预算冻结期内被驳回，则允许用户基于**原单**重新发起，但需满足以下规则：

1. **仅允许原单重提，不属于用户主动新建工单**
2. **系统生成的工单号为新的工单号，不能沿用原单的工单号**
3. **新工单需保留与原工单的关联关系，用于识别其来源于原单重提**
4. **预算号必须沿用原预算号，不允许切换预算号**
5. **重提时仍按该预算当前剩余额度（净额）进行校验**
6. **该特例仅在预算到期前有效**
7. **预算到期后，不再允许继续基于原单重提使用该预算号**

### 10.3 冻结期重提场景的前端交互要求

在“驳回后重新发起”场景中：

1. 若原预算已进入申请冻结期但尚未到期：
   - 允许沿用原预算重新发起
   - 预算号需锁定，不允许修改
   - 预算选择入口应禁用或不可进入
   - 需明确提示：
     - 当前预算已进入申请冻结期，仅允许沿用原预算重新发起，不可更换预算号

2. 若原预算已到期：
   - 不允许继续基于原单重提
   - 需明确提示：
     - 原预算已到期，当前不可继续基于原单重新发起，请联系相关人员处理

## 十二、开发与验收要点

### 12.1 开发侧要点
1. 调整预算卡片展示字段与视觉层级；
2. 实现预算标签计算与单标签显示逻辑；
3. 实现预算过滤、排序与零额度后置逻辑；
4. 实现申请冻结中预算在普通新发起与原单重提场景中的差异化处理；
5. 实现不可选态与空态文案反馈；
6. 保持预算选择页仅支持预算号搜索的既有结论不变。

### 12.2 验收侧要点
重点验收以下内容：

1. 预算卡片中“剩余额度（净额）”是否成为最优先信息；
2. 预算号是否已弱化；
3. 预算总额、归属组织、使用范围是否正确展示；
4. 标签是否只显示一个且优先级正确；
5. 零剩余额度预算是否排后且不可选；
6. 草稿、已作废、已到期预算是否已隐藏；
7. “申请冻结中”预算在普通新发起与驳回后重提场景中的行为是否正确；
8. 原单重提是否为新工单号且预算号锁定；
9. 搜索结果、空态与不可选提示是否符合规则。
