## 变更背景

详情页订单结果区当前无法表达以下业务场景：

1. **订单生成失败原因类型**：需明确列出"产品库存不足"、"专卖店下单标识关闭"、"订单审核驳回草稿"三类
2. **专卖店变更**：工单记录的专卖店（A）与最终生成订单的专卖店（B）不一致时，需展示实际订单所属专卖店
3. **订单备注**：多次生成订单、更换专卖店等操作需显示说明备注

## 变更内容

### 1. 类型定义扩展

**文件**：`vue-reference/src/types/index.ts`

`RelatedOrder` 接口新增字段：

```typescript
export interface RelatedOrder {
  orderNo: string;
  orderType: '产品申请表订单' | '内部申请表订单';
  orderStatus: string;
  storeCode?: string;    // 实际订单所属专卖店编号
  storeName?: string;    // 实际订单所属专卖店名称
  remark?: string;       // 订单备注（如专卖店变更说明）
}
```

**判定规则**：
- `storeCode`/`storeName` 可选：仅当实际订单专卖店与工单明细专卖店不一致时填写
- `remark` 可选：仅当有特殊说明时需要填写（如"原专卖店 A 库存不足，由专卖店 B 承接"）
