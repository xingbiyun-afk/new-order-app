## 一、变更背景

产品申请工单已在 `new-order-app` React 技术栈中完成 `CR-20260629-001` 的 P0 + P1 全部功能实现（含发起页、详情页、我的列表页等 8 个页面）。然而，目标 App 实际采用 Vue3 技术栈开发，React 参考实现无法直接被 Vue 工程复用。

因此，本次变更的核心目标是将已验证通过的 React 参考实现**完整迁移为 Vue3 + TypeScript + Vite 参考实现**，放置于 `vue-reference/` 目录下，供后续 Vue 正式工程直接参考样式、交互、结构与状态管理方案。

### 1.1 迁移范围

| 维度 | React 参考实现（源） | Vue 参考实现（目标） |
|------|---------------------|---------------------|
| 框架 | React 18 | Vue 3.5 |
| 路由 | React Router（Hash） | Vue Router 4（Hash） |
| 状态管理 | React Context + useState | Pinia |
| 构建工具 | Vite 6 | Vite 8 |
| 样式 | Tailwind CSS 3 + 自定义 CSS | Tailwind CSS 4 + 自定义 CSS |
| 类型系统 | TypeScript | TypeScript |

### 1.2 变更来源

- **源实现**：`CR-20260629-001` — 产品申请工单 React 参考实现（P0 + P1）
- **开发任务说明**：`docs/product/work-order-budget/产品申请工单AI开发任务说明.md`

### 5.1 Pinia Store 结构

**Store**：`src/stores/workOrder.ts`

| State | 类型 | 说明 |
|-------|------|------|
| `selectedBudget` | `Budget \| null` | 当前选中的预算 |
| `storeGroups` | `StoreGroup[]` | 专卖店分组数组 |
| `collapsedGroups` | `Set<string>` | 折叠状态的分组 ID 集合 |

| Getter | 说明 |
|----------|------|
| `totalAmount` | 全部分组金额合计 |
| `skuTotalMap` | Map<productCode, totalQuantity> 跨分组 SKU 累计 |

| Action | 说明 |
|--------|------|
| `selectBudget(id)` | 选中预算 |
| `toggleGroupCollapse(id)` | 切换分组折叠 |
| `addGroup()` / `deleteGroup(id)` | 添加 / 删除分组 |
| `updateStoreGroup(gid, code, name)` | 更新分组专卖店 |
| `updateProduct(gid, pid, updates)` | 更新产品属性 |
| `addProduct(gid)` / `deleteProduct(gid, pid)` | 添加 / 删除商品 |
| `initStoreGroups()` | 从 sessionStorage 恢复分组 |
| `persistStoreGroups()` | 保存分组到 sessionStorage |

### 5.2 跨页面数据传递

延续 React 参考实现方案：**URL Search Params**

| 方向 | 参数 | 说明 |
|------|------|------|
| 预算选择 → 发起页 | `budgetId` | 回填预算信息 |
| 专卖店选择 → 发起页 | `storeCode` / `storeName` / `groupId` | 回填专卖店 |
| 产品选择 → 发起页 | `productId` / `productCode` / `groupId` | 回填产品 |

发起页通过 `watch(() => route.query, ...)` 监听参数变化，回填完成后 `router.replace({ query: {} })` 清除参数。

## 八、Bug 修复记录

### 修复1：CDN 部署 chunk 加载问题

**问题**：懒加载路由（`import()`）在 CDN 部署环境下 chunk 路径解析失败，页面空白。  
**修复**：将路由组件从动态 `import()` 改为同步 `import`，打包为单一 bundle。  
**提交**：`47875bc`

### 修复2：专卖店/产品选择回填失败

**问题**：从选择页返回发起页时，URL 参数未正确回填到表单。  
**根因**：`route.query` 通过 `computed` + `watch` 间接监听无法触发响应式更新。  
**修复**：改为直接 `watch(() => route.query, ...)` + `immediate: true`，确保组件挂载时立即检查。  
**提交**：`e424445`
