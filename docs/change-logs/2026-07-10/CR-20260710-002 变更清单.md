## 2. 变更背景

`CR-20260710-001` 已完成第一轮产品文档治理，当前已收口以下基础能力：

1. 产品目录保留文档命名已基本中文化
2. 主文档 / 页面规则文档 / 辅助文档 / 协作文档层级已建立
3. 产品文档总览入口已建立
4. README 已从堆叠式目录说明调整为分层式入口
5. 小体量、边界弱的小文档已完成并入、降级或归档处理

但当前文档体系仍存在以下问题：

1. **PRD 偏厚，局部仍承接过多页面级细节**
2. **页面结构说明与页面规则文档之间存在一定重复**
3. **字段清单与页面规则文档、PRD 之间存在局部重复叙述**
4. **预算核销规则文档体量偏薄，尚未达到“专项主文档”的完整度**
5. **部分重点文档缺少统一的阅读提示、适用边界、冲突优先级、引用说明**
6. **当前目录虽已清晰，但文档内容层面的阅读成本仍有进一步下降空间**

因此，本轮需要在不改业务规则的前提下，继续对重点文档进行 **内容收敛、结构优化、层级强化与可读性提升**。

---

## 3. 本轮变更目标

本轮目标不是新增规则，而是让“已经收口的规则”变得更清楚、更好读、更好用。

本轮需完成以下 6 项目标：

1. **PRD 瘦身但不失主线**
2. **页面结构说明文档降重、回归结构总览定位**
3. **字段清单文档去重、回归字段定义定位**
4. **预算核销规则文档补强为稳定专项规则文档**
5. **主文档统一补充阅读入口、边界与引用说明**
6. **保证保留文档之间“谁是上游、谁是页面明细、谁是辅助说明”关系更清晰**

## 5. 本轮纳入范围

### 5.1 PRD 内容结构优化
对 PRD 执行“主线保留、细节下沉、重复压缩、阅读引导增强”的优化。

### 5.2 页面结构说明文档瘦身
将页面结构草图说明文档进一步收口为：

- 结构原则
- 区块组织
- 页面关系
- 模型示意
- 页面细节引用入口

不再承担过多页面细则描述。

### 5.3 字段清单文档去重
将字段清单文档收口为：

- 字段定义
- 来源
- 校验
- 展示属性
- 与页面规则文档的分工说明

减少流程、结构、交互类重复描述。

### 5.4 预算核销规则文档补强
将预算核销规则文档补强为“预算核销专项规则主文档”，但不扩写为后台系统设计文档。

### 5.5 主文档统一补阅读入口
对重点主文档补充统一化开头说明，包括但不限于：

1. 文档定位
2. 适用范围
3. 本文重点解决什么问题
4. 本文不解决什么问题
5. 与其他文档冲突时以谁为准
6. 推荐阅读顺序或上游引用说明

### 5.6 引用链与层级说明优化
对保留文档之间的引用语句和边界描述做统一化修正，避免：

- “既像主文档又像辅助文档”
- “同一问题多个文档都在重复定义”
- “读者看不出该先看哪一份”

## 7. 本轮执行原则

### 7.1 规则不变，表达更好
本轮原则是：

> **不改变规则，只优化表达。**

### 7.2 主文档负责主线，页面文档负责细则
- PRD 负责：
  - 背景
  - 目标
  - 范围
  - 主业务链路
  - 核心规则
  - 总体结构
- 页面规则明细文档负责：
  - 页面级执行细则
- 辅助文档负责：
  - 字段
  - 页面结构总览
- 协作文档负责：
  - 开发约束
  - AI 执行边界

### 7.3 去重不等于删结论
凡从某文档中弱化或移除的内容，必须满足：

1. 该结论已在更合适的主文档中稳定存在
2. 移除后不影响读者理解
3. 移除后必须保留“详见某文档”的引用引导

### 7.4 优化后必须更易读
本轮验收不是看“删了多少”，而是看：

1. 阅读顺序是否更清楚
2. 文档角色是否更清楚
3. 重复是否明显下降
4. 重点是否更容易找到
5. 文档之间是否更少打架

- 是否明显减少细节性规则重复

### 12.4 字段清单是否回归字段定义定位
- 是否明显减少流程和交互重复

- 是否能帮助不同角色快速找到该看的文档

- 本轮只改表达，不改规则
- 不得引入新口径
- 不得回退旧术语
