## 6. 页面实现核心规则

## 6.1 发起页
发起页需按以下顺序组织：

1. 预算信息区
2. 申请基础信息区
3. 产品申请订单明细区（含产品明细）
4. 附件区
5. 提交区

> **说明**：
- 每个专卖店申请分组内包含订单归属专卖店和多条商品明细。同一分组下先选专卖店，再添加商品明细。商品明细不再作为独立一级区块，而是归入专卖店申请分组内。
- 本文部分业务解释中如出现“分组”，指内部结构概念；发起页用户侧 UI 统一展示为“产品申请订单明细”。

### 6.1.1 发起页重点要求
- 预算号可选择；
- 预算异常提示仅在发起页展示；
- 申请人、申请类型、申请理由自动带出；
- 专卖店申请分组支持多个，每个分组下产品明细支持多条；
- 数量可编辑，金额自动计算；
- **发起页不单独展示折扣字段**，仅展示“是否打折产品：是 / 否”；
- 订单归属专卖店通过搜索选择页选择；
- **已选专卖店支持重新选择和清空**（清空时同步清空该分组商品明细）；
- **切换预算时，若已填写分组或商品，须弹出确认提示**；确认后清空现有分组和商品明细，再完成预算切换；
- 提交前校验整单金额不超过预算可用金额；
- 附件支持上传区占位、预览、删除交互参考；
- 发起页产品申请订单明细区提供**整单级批量导入**入口（`批量导入产品`），与`添加产品申请订单`并列；
- 批量导入支持以下方式：复制粘贴识别导入（逗号/空格/制表符分隔）、Excel/CSV 文件导入；
- 导入数据固定字段顺序：产品编号、数量、专卖店编号；
- 导入校验规则：产品不在核销范围/专卖店不可选/数量非正整数 → 硬拦截（全有或全无）；数量超限 → 允许导入但提交拦截；
- 导入成功后按专卖店编号自动拆分为多个产品申请订单明细；
- 若页面已有产品申请订单明细数据，执行导入前需弹确认提示（覆盖导入）；
- 提供 CSV 模板下载功能；
- 仅保留提交按钮，不做保存草稿按钮。

### 6.1.2 数量与金额规则
- 同一商品数量需受可申请数量控制；
- 页面先做前置控制；
- 提交时仍以后端库存校验为准；
- 金额自动汇总，不允许手工输入；
- 打折产品金额按“产品申请管理菜单维护的折扣”计算；
- 当前参考数据可用 50%（5.0折），但实现不得写死为不可配置常量。

## 6.2 预算选择页（CR-20260702-001 更新）
预算选择页以卡片列表方式承载，每张卡片信息层级如下：

**展示字段**：
- 预算标签（最多1个，四种类型见下方）
- 预算号（弱化显示，小字号灰色）
- 剩余额度（净额）—— 大字号绿色，卡片视觉焦点
- 预算总额（净额）—— 小字号灰色辅助
- 预算归属组织
- 预算使用范围（纯组织架构口径，不含店铺类型）
- 预算到期日

**不展示**：
- 已用金额、申请中金额
- 产品编号

**预算标签机制**：
| 优先级 | 标签 | 触发条件 |
|--------|------|----------|
| 1 | 最近申请 | 用户最近一次成功提交对应的预算 |
| 2 | 15天内即将到期 | 距离到期日 ≤ 15 天 |
| 3 | 本季度可申请 | 不在冻结期、无更高优先级标签 |
| 4 | 申请冻结中 | freezeStartDate ≤ 今天 < budgetExpiryDate |

- 同一卡片只展示1个最高优先级标签；
- 标签仅给剩余额度 > 0 的预算展示。

**排序规则**：
1. 前置过滤：草稿/已作废/已到期不展示
2. 零剩余额度后置到列表末尾
3. 按标签优先级升序
4. 按到期日升序（近的先）
5. 兜底：剩余额度降序 → 预算号升序

**搜索规则**：
- **仅支持预算号模糊搜索**；
- 搜索占位文案为“搜索预算号”；
- 搜索无结果时展示“搜索无结果”空态。

**禁用与提示规则**：
- `applicationAvailable=false` 时禁用不可选；
- `isAbnormal=true` 但预算可见时，允许选择，同时给提示；
- **零剩余额度**：卡片置灰不可选，点击提示“剩余额度不足”；
- **冻结期预算**（普通新发起）：卡片置灰不可选，显示“申请冻结中”标签，点击提示“已进入申请冻结期”；
- **驳回后重提**：不进入选择页，预算号由系统锁定直接回填发起页。

**交互规则**：
- 卡片整体可点击；
- 可选预算点击后进入选中态（青绿色边框 + 已选标识 ✓），再次点击取消；
- 不可选预算降低透明度，叠加遮罩层显示不可选原因；
- 选中后底部展示“确认选择”按钮，点击返回发起页并联动带出申请类型、申请理由。

**空态**：
- “当前无可申请预算”——无搜索词时列表为空
- “搜索无结果”——有搜索词但未匹配到预算号

> **详细规则**：预算选择页的字段展示、标签计算、可选性判定、排序、冻结期重提等页面级详细规则，以 `docs/product/work-order-budget/产品申请工单选择预算页面规则明细.md` 为准。

## 7. 关键业务规则摘要

### 7.1 预算规则
- 提交进入审批流时占用预算
- 审批中金额进入申请中金额
- 驳回释放申请中金额
- 审批通过后转为已用金额
- 审批通过即视为预算核销完成
- 后续订单失败不自动回滚预算核销

### 7.2 功能订单规则
- 工单提交进入审批流后，生成功能订单预占库存
- 功能订单不产生交易
- 审批通过或驳回后取消功能订单并释放库存

### 7.3 订单生成规则
- 客户层级 = 6 → 产品申请表订单
- 客户层级 = 0-4 → 内部申请表订单
- 层级 5 不允许作为订单归属专卖店选择
- 订单生成失败时，订单流程回退为草稿状态
- 工单审批结果不回退
- 预算核销不自动回滚

### 7.4 客户与专卖店规则
- 客户数据来源于客户信息表
- 订单归属专卖店采用“编号 + 名称”展示
- 搜索时支持客户编号 / 客户名称
- 默认只能选择所属组织管辖范围内、本人有数据权限、且非层级 5 的客户编号

### 7.5 产品与折扣规则
- 产品可选范围、JDE价格、是否打折产品、折扣值、可申请数量，均来源于产品申请管理菜单维护；
- 当前业务参考折扣值为 50%（5.0折），但系统允许后续调整；
- **发起页、详情页不单独展示折扣字段**，只展示“是否打折产品：是 / 否”；
- **产品选择页作为辅助选品页，可展示折扣信息**（如“折扣：5.0折”），帮助用户理解价格与金额规则；
