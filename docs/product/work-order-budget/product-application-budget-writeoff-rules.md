# 产品申请工单预算核销规则

## 1. 入口名称

入口名称统一为：产品申请工单。

历史“预算申请工单”仅为旧口径参考。

## 2. 预算核销时点

- 提交产品申请工单进入审批流时，占用预算；
- 审批中，该金额进入申请中金额；
- 驳回后，释放申请中金额；
- 审批通过后，申请中金额转为已用金额；
- 审批通过即预算核销完成；
- 后续订单失败不自动回滚预算核销。

## 3. 金额字段来源

正式实现以后端返回为准。

建议字段：

- 预算金额
- 已用金额
- 申请中金额
- 可用金额

Mock 阶段可参考：

可用金额 = 预算金额 - 已用金额 - 申请中金额

但必须注明正式实现以后端返回为准。

## 4. 审批路径匹配规则

当前已确认审批路径匹配依据：

1. 发起人组织身份
2. 预算归属组织
3. 预算适用范围

不得自行加入金额区间、产品类型、费用类型、客户类型等条件。

## 5. 功能订单预占库存

- 产品申请工单提交进入审批流后，同时生成对应功能订单；
- 功能订单用于预占库存，不产生交易；
- 审批中，功能订单持续占用库存；
- 审批通过或驳回后，功能订单取消，库存释放。

## 6. 订单生成规则

审批通过后，根据客户层级生成订单：

- 客户层级 6：产品申请表订单
- 客户层级 0-4：内部申请表订单

该规则为现有系统成熟固定规则，本次只需衔接。

## 7. 订单生成失败处理

- 订单生成失败时，订单回退草稿状态；
- 必须展示失败原因；
- 允许修改草稿后重新发起；
- 允许从现有产品申请表订单或内部申请表订单新建草稿；
- 新建草稿时可关联审批通过的产品申请工单号；
- 订单失败不代表产品申请工单审批回退；
- 订单失败不代表预算核销自动回滚。

## 8. 废弃概念

不得使用：

- 预算责任人
- 主责 / 备责
- 待调整
- 责任人异常影响预算核销
- 预算核销资格名单管理
