## 1. 文档说明

### 1.1 文档定位
本文档用于沉淀“产品申请工单”发起链路中**选择预算页面**的页面规则、展示规则、标签规则、可选性规则、排序规则与冻结期重提特例规则，作为后续：

- Vue 参考实现维护
- 页面验收
- 后续正式开发承接
- 多 AI 协作时的预算选择页统一依据

### 1.2 适用页面
本文档适用于：

1. 产品申请工单发起页中的“选择预算”页面；
2. 发起页中已选预算摘要区的预算信息承接；
3. 驳回后重新发起场景下，与预算锁定相关的前端交互承接。

### 1.3 文档边界
本文档仅描述**预算选择页面相关规则**，不重述以下完整业务内容：

- 预算核销主流程
- 工单整体发起规则
- 产品申请订单明细规则
- 预算管理后台维护规则全量细节

如与上层 PRD 或预算维护规则文档冲突，以最新确认的产品申请工单业务口径为准。

边界说明：本文档仅描述预算选择页面及其与发起页预算摘要承接相关的规则，不展开创建页主体填写规则。创建页初始化、申请信息区、产品申请订单明细、产品选择、批量导入、附件与提交状态等规则，详见《产品申请工单创建页面规则明细.md》。

## 2. 页面目标

选择预算页面的核心目标不是展示预算管理全量信息，而是帮助申请人在移动端场景下快速完成以下判断：

1. 当前有哪些预算可以用于产品申请；
2. 哪条预算的**剩余额度（净额）**最值得优先关注；
3. 某条预算是本人组织预算，还是总部 / 区域 / 省办授权可用预算；
4. 哪些预算最近用过、即将到期、当前处于申请冻结期；
5. 哪些预算不可选，以及为什么不可选。

因此，本页面的信息组织原则为：

- **优先帮助选择**
- **弱化后台管理视角**
- **突出剩余额度**
- **控制卡片阅读负担**
- **避免用户误选不可用预算**

## 3. 页面入口与适用场景

### 3.1 普通新发起场景
用户在产品申请工单发起页中点击“预算号”后，进入选择预算页面，选择一条预算并回填至发起页。

### 3.2 驳回后重新发起场景
若当前为**原单重提**场景，则不应按普通新发起逻辑重新开放预算选择。

规则如下：

1. 原单重提不属于用户主动新建工单；
2. 原单重提后，**系统生成的工单号为新的工单号，不能沿用原单的工单号**；
3. 新工单仍需保留与原工单的关联关系，用于识别其来源于原单重提；
4. 若原预算处于申请冻结中但尚未到期：
   - 可沿用原预算重新发起；
   - 预算号必须锁定；
   - 不允许重新进入预算选择页面切换预算；
5. 若原预算已到期：
   - 不允许继续基于原单重提；
   - 页面需明确提示联系相关人员处理。

说明：  
当前参考实现中，驳回后重提完整数据流可采用 mock / 占位方式承接，但前端交互限制与文案口径需按本规则执行。

## 4. 预算展示范围规则

### 4.1 基础展示范围
预算选择页面仅展示**用户权限范围内可见**的预算。

页面不得突破现有权限控制，不得展示超出用户可见范围的预算。

### 4.2 组织可见性补充规则
结合《季度预算维护页面规则明细》，预算在选择页面中的可见性除基础权限控制外，还需遵循以下归属组织与适用范围规则：

#### 1）区域模式
若预算归属为**区域**：

- 仅该区域层级员工可见 / 可发起 / 可使用；
- 所属省办员工不可见不可用。

#### 2）省办模式
若预算归属为**省办**：

- 省办员工可见 / 可发起 / 可使用；
- 其所属区域层级员工也可见 / 可发起 / 可使用。

#### 3）总部归属（市场部 / 销售运作部）且适用范围为空
若预算归属为**市场部（MKT）**或**销售运作部（TMK）**，且未配置适用范围：

- 仅归属部门本身可见 / 可发起 / 可使用。

#### 4）总部归属（市场部 / 销售运作部）且适用范围为区域或省办
若预算归属为**市场部（MKT）**或**销售运作部（TMK）**，且适用范围配置为区域或省办：

- 仅被授权区域 / 省办范围内用户可见 / 可发起 / 可使用；
- 原归属部门用户在产品申请工单发起选择预算号时**不再可见**。

示例：
- 预算归属组织为市场部；
- 适用范围配置为“华东区”或“江苏省办”；
- 则市场部同事本人在申请侧不可看到该预算；
- 仅被授权范围内的区域 / 省办用户可以看到并使用。

## 5. 状态过滤规则

### 5.1 不展示的预算范围
以下预算不在选择预算页面展示：

1. 草稿
2. 已作废
3. 已到期

说明：  
本页面是申请动作页面，不是预算管理查询页面，因此仅保留对申请行为有意义的预算。

### 5.2 零剩余额度预算展示规则
对于用户权限范围内可见、但**剩余额度（净额）为 0** 的预算：

1. 允许继续展示；
2. 但需整体排在可用预算之后；
3. 需采用弱化样式；
4. 明确不可选。

该规则用于帮助用户识别历史可见预算，但不应让零额度预算抢占主列表视觉重点。

## 6. 卡片展示字段规则

### 6.1 核心展示原则
预算卡片应围绕“是否值得选、是否现在可用”进行信息组织。

### 6.2 卡片展示字段
预算卡片展示以下字段：

1. 预算号
2. 剩余额度（净额）
3. 预算总额（净额）
4. 预算归属组织
5. 预算使用范围
6. 标签（最多 1 个）
7. 异常提示（如存在）

### 6.3 字段展示优先级
字段展示优先级如下：

#### 第一优先级
- **剩余额度（净额）**

要求：
- 成为卡片第一视觉重点；
- 金额颜色、字号和位置均需高于其他字段；
- 便于用户第一眼判断“还能不能用”。

#### 第二优先级
- 标签
- 预算总额（净额）

#### 第三优先级
- 预算归属组织
- 预算使用范围

#### 弱化信息
- 预算号

### 6.4 字段命名规则
本页面字段命名统一如下：

- `可用金额` → **剩余额度（净额）**
- 新增：**预算总额（净额）**
- 新增：**预算归属组织**
- 新增：**预算使用范围**

本页面不再展示：

- 已用金额
- 申请中金额

## 7. 预算归属组织与预算使用范围展示规则

### 7.1 总体原则
预算维护页允许使用组织 ID、归属组织类型、适用范围摘要等配置口径；  
但在申请侧选择预算页面中，应转换为**用户可读的业务展示内容**，不得直接展示后台组织 ID。

用户侧不得直接展示以下内部值：

- `MKT`
- `TMK`
- `R001`
- `P001`
- `HQ_PARENT`

尤其：
- `HQ_PARENT` 仅为系统内部父级组织标识，不得出现在用户侧。

### 7.2 预算归属组织显示规则
预算归属组织用于展示预算实际归属主体。

#### 示例
1. 归属组织类型 = 总部，归属组织 = `MKT`
   - 显示：**市场部**
2. 归属组织类型 = 总部，归属组织 = `TMK`
   - 显示：**销售运作部**
3. 归属组织类型 = 区域，归属组织 = 华东区
   - 显示：**华东区**
4. 归属组织类型 = 省办，归属组织 = 江苏省办
   - 显示：**江苏省办**

### 7.3 预算使用范围显示规则
预算使用范围用于展示该预算当前允许哪些组织使用。

#### 规则 1：总部归属且适用范围为空
若预算归属为市场部 / 销售运作部，且适用范围未配置区域或省办：

- 页面不显示“空”
- 应显示为归属部门本身

#### 示例
- 预算归属组织：市场部
- 预算使用范围：**市场部**

- 预算归属组织：销售运作部
- 预算使用范围：**销售运作部**

#### 规则 2：总部归属且适用范围为区域或省办
若预算归属为市场部 / 销售运作部，且适用范围配置为区域或省办：

- 预算归属组织仍显示归属部门；
- 预算使用范围显示被授权的区域 / 省办摘要。

#### 示例
- 预算归属组织：市场部
- 预算使用范围：**华东区**

- 预算归属组织：市场部
- 预算使用范围：**江苏省办**

- 预算归属组织：市场部
- 预算使用范围：**华东区、华南区**

- 预算归属组织：市场部
- 预算使用范围：**江苏省办、浙江省办等 4 个**

#### 规则 3：区域归属
若预算归属为区域：

- 预算归属组织显示该区域；
- 预算使用范围默认显示该区域本身；
- 如页面采用摘要增强，也可显示为：
  - `华东区（全区域）`

#### 规则 4：省办归属
若预算归属为省办：

- 预算归属组织显示该省办；
- 预算使用范围默认显示该省办本身。

### 7.4 摘要折叠规则
预算使用范围的摘要折叠规则与预算维护页保持一致：

1. 1~3 项时，直接展示具体组织名称；
2. 4 项及以上时，采用“等 N 个”方式折叠展示。

#### 示例
- 江苏省办、浙江省办、安徽省办
- 江苏省办、浙江省办等 4 个

### 7.5 页面展示压缩规则
考虑移动端卡片空间限制：

1. 预算归属组织默认单行展示；
2. 预算使用范围采用摘要式展示；
3. 超长内容允许截断；
4. 本轮不强制要求在选择预算页内新增完整展开弹层；
5. 若需查看完整范围，可在后续预算详情页或创建页规则中进一步扩展。

## 8. 卡片样式与阅读规则

### 8.1 预算号样式
预算号用于识别，不作为卡片主信息。

要求：

1. 缩小字号；
2. 降低视觉权重；
3. 不与金额抢占同一视觉层级。

### 8.2 剩余额度样式
剩余额度（净额）需强化展示：

1. 颜色醒目；
2. 字号明显大于普通字段；
3. 与“预算总额（净额）”形成主辅层级。

### 8.3 异常提示展示规则
若预算存在组织异常提示（`isAbnormal=true`）：

1. 允许继续展示预算卡片；
2. 以提示条形式展示异常说明；
3. 该提示仅用于提醒，不直接阻断预算使用。

说明：  
`isAbnormal` 仅表示组织异常提示，不等于预算不可用。

## 9. 预算标签规则

### 9.1 标签目标
预算标签用于帮助用户快速识别更值得优先关注的预算，不替代预算状态本身。

### 9.2 标签列表

#### 1）最近申请
满足以下条件时显示：

- 在用户权限范围内可见；
- 为用户最近一次**提交成功**的产品申请工单所使用的预算；
- 当前剩余额度（净额）大于 0。

说明：
- 仅以“提交成功”作为判定依据；
- 当前业务不以草稿作为“最近申请”的计算口径。

#### 2）15天内即将到期
满足以下条件时显示：

- 在用户权限范围内可见；
- 距离 `budgetExpiryDate` 小于等于 15 天；
- 当前剩余额度（净额）大于 0。

#### 3）本季度可申请
满足以下条件时显示：

- 在用户权限范围内可见；
- 已进入预算期间可申请范围；
- 当前剩余额度（净额）大于 0。

说明：  
参考实现阶段可使用 mock 近似实现；正式环境以后端预算期间字段或正式规则为准。

#### 4）申请冻结中
满足以下条件时显示：

- 在用户权限范围内可见；
- 已进入 `freezeStartDate`；
- 当前时间尚未达到 `budgetExpiryDate`；
- 当前剩余额度（净额）大于 0。

说明：  
该标签用于表达“已进入新增申请冻结期，但尚未正式到期”的预算。

### 9.3 标签展示数量规则
同一卡片**只显示 1 个标签**。

若同时命中多个标签，仅显示优先级最高的那个。

### 9.4 标签优先级
标签优先级如下：

1. 最近申请
2. 15天内即将到期
3. 本季度可申请
4. 申请冻结中

## 10. 预算可选性规则

### 10.1 可选预算
预算同时满足以下条件时，前端可允许选择：

1. 在用户权限范围内可见；
2. 状态为已生效；
3. `applicationAvailable = true`
4. 剩余额度（净额）大于 0；
5. 当前不处于普通新发起场景下的申请冻结限制。

### 10.2 不可选预算
以下任一情况成立时，预算前端不可选：

1. `applicationAvailable = false`
2. 剩余额度（净额）为 0
3. 当前处于申请冻结中，且场景为普通新发起
4. 其他超出当前页面可申请口径的情况

### 10.3 不可选预算提示规则
不可选预算不应“看似可点但无反馈”。

点击后应给出清晰提示：

#### 1）`applicationAvailable = false`
提示示例：
- 当前预算不可用于申请

#### 2）剩余额度（净额）为 0
提示示例：
- 当前预算剩余额度不足，无法用于申请

#### 3）申请冻结中
提示示例：
- 当前预算已进入申请冻结期，不可用于新发起

---

## 11. 排序规则

### 11.1 一级排序：标签优先级
预算列表优先按标签优先级排序：

1. 最近申请
2. 15天内即将到期
3. 本季度可申请
4. 申请冻结中

### 11.2 二级排序：距离到期日
在标签优先级基础上，再按距离 `budgetExpiryDate` 的时间升序排序：

- 越接近到期日，越靠前

### 11.3 零剩余额度预算后置
剩余额度（净额）为 0 的预算整体排在后面。

### 11.4 兜底排序
若标签优先级与到期时间均相同，按以下顺序继续排序：

1. 剩余额度（净额）降序
2. 预算号升序

## 12. 搜索规则

### 12.1 搜索范围
预算选择页面仅支持：

- **预算号搜索**

### 12.2 搜索结果规则
搜索结果仍需遵循以下规则：

1. 仅返回用户权限范围内可见预算；
2. 不打破状态过滤规则；
3. 不打破不可选规则；
4. 不打破已定义的排序规则。

### 12.3 搜索无结果
若按预算号搜索未命中结果，应展示“搜索无结果”空态，不得与“当前无可申请预算”空态混用。

---

## 13. 预算数据时效规则

### 13.1 进入选择预算页面时
用户每次进入选择预算页面时，看到的预算数据应为**当前实时数据**。

包括但不限于：

- 剩余额度（净额）
- 预算状态
- 是否可申请
- 标签计算所依赖的时间数据

### 13.2 选择后返回发起页时
用户在选择预算并返回产品申请工单发起页后，发起页中的预算摘要区**不做持续实时刷新**，仅保留用户选择当时的回填结果用于填写过程展示。

### 13.3 提交申请时
在提交产品申请工单时，仍需基于预算号实际最新数据再次校验是否可以提交成功。

包括但不限于：

- 是否仍可申请
- 剩余额度是否足够
- 是否已进入不可申请状态
- 其他以后端实时校验为准

说明：  
关于提交前预算校验、更详细的创建页时效规则与失败反馈，放在后续《创建产品申请工单页面规则明细》文档中进一步细化。

## 14. 交互反馈规则

### 14.1 卡片点击范围
预算卡片整体应支持点击，不依赖局部微小点击区域。

### 14.2 选中态
选中预算后，卡片需有明确选中态，例如：

- 边框高亮
- 背景变化
- 已选标识

### 14.3 不可选态
不可选预算应有明显弱化表现，例如：

- 降低透明度
- 弱化背景
- 保留原因提示能力

### 14.4 空态区分
页面需区分以下两类空态：

#### 1）当前无可申请预算
适用于按权限、状态和规则过滤后无预算可展示。

#### 2）搜索无结果
适用于用户已输入预算号关键词但未命中任何结果。

两类空态文案与图示不应混用。

## 15. 冻结期与原单重提特例

### 15.1 普通新发起限制
根据预算维护规则：

- `freezeStartDate` 默认取 `endDate + 45天`
- 自 `freezeStartDate` 起，不可再发起新的产品申请工单用于预算核销

因此，在普通新发起场景下：

- 进入申请冻结中的预算不可选

### 15.2 原单重提特例
若某张已使用该预算号的产品申请工单，在预算冻结期内被驳回，则允许用户基于**原单重提**重新发起，但需满足：

1. 不属于用户主动新建工单；
2. 系统生成的工单号为新的工单号，不能沿用原单的工单号；
3. 新工单需保留与原工单的关联关系；
4. 预算号必须沿用原预算号，不允许切换；
5. 仍需按当前剩余额度（净额）进行校验；
6. 仅在预算到期前有效；
7. 预算到期后，不再允许继续基于原单重提使用该预算号。

### 15.3 前端承接要求
在发起页中：

#### 原预算冻结中但未到期
- 允许沿用原预算重新发起；
- 预算号锁定；
- 预算选择入口禁用；
- 提示：
  - 当前预算已进入申请冻结期，仅允许沿用原预算重新发起，不可更换预算号

#### 原预算已到期
- 不允许继续基于原单重提；
- 提示：
  - 原预算已到期，当前不可继续基于原单重新发起，请联系相关人员处理

## 16. 发起页预算摘要区承接规则

选择预算完成后，发起页中的预算摘要区应至少回显以下字段：

1. 预算号
2. 剩余额度（净额）
3. 预算总额（净额）
4. 预算归属组织
5. 预算使用范围
6. 预算异常提示（如存在）

不再回显：

- 已用金额
- 申请中金额
