# 产品申请工单字段清单初稿（v0.2）

## 1. 文档说明

本文档用于补充《产品申请工单和产品申请预算核销PRD》中与页面字段直接相关的定义，作为 App 端 Vue 参考实现阶段的字段依据。

当前字段清单按以下 4 个视角整理：

1. 发起页字段
2. 列表卡片字段
3. 详情页字段
4. 审批操作区字段

说明：

- 本文档以当前已确认业务规则为准；
- 未完全明确的字段，以“待补充”或“实现占位”标识；
- 当前目标是支撑 AI 开始 App 初版参考开发，不等同于最终生产字段协议。

---

## 2. 发起页字段

### 2.1 发起页整体说明
发起页采用现有工单表单式交互承接，字段以“行式表单 + 点击选择 + 自动带出 + 局部编辑”为主。

当前发起页需至少覆盖：

- 预算信息
- 申请基础信息
- 产品明细
- 订单归属专卖店信息
- 附件上传
- 提交动作

### 2.3 折扣规则
1. 折扣值来源于**产品申请管理菜单**维护；
2. 当前业务已确定的现行配置值为 **50%（5.0折）**；
3. 系统支持业务根据实际情况调整折扣值；
4. 折扣不由发起人在工单页面填写，也不允许在工单页面修改；
5. 页面为节省移动端展示空间，不单独展示折扣字段，仅展示“是否打折产品：是 / 否”；
6. 金额计算时，以产品申请管理菜单中维护的实际折扣值为准。

### 2.4 产品明细子表规则补充

#### 2.4.1 多条明细
产品申请工单支持多条产品明细。

#### 2.4.2 单条明细金额计算
- 打折产品：  
  `金额（净额） = JDE价格 × 数量 × 折扣`
- 不打折产品：  
  `金额（净额） = JDE价格 × 数量`

其中：
- `折扣` 由产品申请管理菜单维护；
- 当前现行业务值为 **50%（5.0折）**；
- 后续允许业务调整。

#### 2.4.3 合计金额规则
- 所有产品明细金额（净额）自动汇总；
- 汇总金额不可手工修改；
- 汇总金额不得超过预算可用金额；
- 提交时需再次校验，防止并发申请导致实际可用金额变化。

#### 2.4.4 可申请数量规则
- 未设置限量：可申请数量跟随可用库存变动；
- 设置限量大于可用库存：按实际可用库存显示；
- 设置限量小于可用库存：按限量显示；
- 限量为 0：表示可申请数量为 0。

#### 2.4.5 数量校验说明
页面录入数量时需先按可申请数量进行前置控制，但工单最终能否成功提交、能否成功生成预占库存订单，仍以后端提交时的实际库存结果为准。

若存在相同 SKU 分配到不同订单归属专卖店的情况，则这些明细行对应数量合计不能超过该 SKU 的可申请数量。

---

### 2.5 订单归属专卖店选择规则补充

#### 2.5.1 选择交互
沿用现有工单交互：

1. 点击表单字段进入搜索页；
2. 搜索页支持输入客户编号或客户名称；
3. 展示结果列表；
4. 选中后返回表单页并回填；
5. 表单中按“编号 + 名称”展示结果。

#### 2.5.2 可选范围
- 区域用户：可选区域范围内（含下属省办）且本人有数据权限、且层级不为 5 的客户编号；
- 省办用户：可选省办范围内且本人有数据权限、且层级不为 5 的客户编号；
- 总部部门用户：可选全国范围内且本人有数据权限、且层级不为 5 的客户编号。

#### 2.5.3 层级限制
层级 5 客户存在，但不允许作为订单归属专卖店被选中。

## 5. 审批操作区字段

### 5.1 审批场景说明
审批操作区在当前用户为待处理审批人，且工单处于可处理状态时显示。

### 5.3 审批操作补充说明
- 审批通过后：
  - 工单审批通过；
  - 预算申请中金额转为已用金额；
  - 功能订单取消并释放库存；
  - 进入正式订单生成流程。

- 审批驳回后：
  - 工单状态为已驳回；
  - 预算申请中金额释放；
  - 功能订单取消并释放库存。

## 7. 当前仍需后续细化的字段项

以下字段已识别，但仍建议后续继续补细：

1. 备注字段最大长度；
2. 金额展示格式是否增加千分位；
3. 发起页预算异常提示的最终文案样式；
4. 附件预览、下载、删除在不同角色下的操作差异；
5. 终审节点中的多条关联订单展示样式，建议采用简化卡片式承载。

已确认规则如下：

- 金额展示保留小数点后两位；
- 折扣按百分比显示；
- 订单归属专卖店在表单中的显示为“编号 + 名称”；
- 发起页附件支持预览、删除；
- 详情页附件支持预览、下载；
- 工单编号不在列表卡片显示；
- 可申请数量不在详情页长期显示。
