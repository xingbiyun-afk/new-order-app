## 2. 超时无人审批场景的监控口径

### 2.1 监控对象
产品申请工单审批中，**任一审批环节**超过规定时间无审批操作的场景。

### 2.2 监控阈值
**14 天**。

> 与系统超时自动驳回阈值保持一致。监控层负责"提前识别与预警"，系统层负责"超时后执行"。

### 2.3 起算点
自**进入当前待审批节点时刻**起算。

### 2.4 监控识别规则
1. 监控需识别：某节点已进入待审批状态
2. 监控需计算：当前时间 - 进入待审批时间
3. 监控需判断：是否已达到或接近 14 天阈值
4. 监控需生成：对应级别的提示或预警

### 2.5 监控提示分级建议
| 级别 | 条件 | 建议动作 |
|------|------|----------|
| 提醒 | 接近 14 天（如 10 天） | 生成待关注提示 |
| 预警 | 达到 14 天前（如 13 天） | 生成即将超时预警 |
| 告警 | 已达到或超过 14 天 | 生成已超时告警 |

### 2.6 监控不执行的动作
1. 监控**不直接驳回**工单
2. 监控**不直接改变**工单状态
3. 监控**不替代**审批人执行审批
4. 监控**不干预**审批流程流转

> 监控仅作为识别与推动依据，最终审批结果由审批人或系统超时规则执行。

## 3. 组织变更后在途工单的监控口径

### 3.1 监控场景
工单已提交且处于审批中，相关组织信息发生变更。

### 3.2 监控识别规则
1. 监控需识别：组织变更事件
2. 监控需关联：变更组织范围下的在途工单
3. 监控需判断：这些工单是否仍处于审批中
4. 监控需生成：组织变更影响在途工单的提示

### 3.3 监控提示内容
监控提示应至少包含：
1. 组织变更事件说明
2. 受影响在途工单数量
3. 建议处理方式（财务通知相关审批人驳回）
4. 超时风险提示（若通知后仍无人处理，将在 14 天后自动驳回）

### 3.4 监控观察口径
1. 监控需持续观察：通知后审批人是否执行驳回
2. 监控需跟踪：从通知到审批的时间间隔
3. 监控需预警：接近 14 天仍未处理的情况

## 4. 监控提示与最终审批结果的边界

### 4.1 核心原则
**监控不直接改变工单结果，只作为识别与推动依据。**

### 4.2 职责边界
| 职责 | 监控层 | 系统层 |
|------|--------|--------|
| 识别超时 | 是 | 是 |
| 生成提示/预警 | 是 | 否 |
| 改变工单状态 | **否** | 是 |
| 执行自动驳回 | **否** | 是 |
| 记录驳回原因 | **否** | 是 |

### 4.3 数据流边界
1. 监控层识别超时 → 生成提示/预警 → 推送给相关人员
2. 系统层判断超时 → 执行自动驳回 → 改变工单状态
3. 两者独立运作，监控层不干预系统层的执行逻辑

### 4.4 展示边界
| 展示内容 | 监控页面 | 工单详情页 |
|----------|----------|------------|
| 超时预警信息 | 是 | 否 |
| 工单最终驳回状态 | 否 | 是 |
| 驳回原因 | 否（仅提示） | 是（最终结果） |
| 组织变更影响提示 | 是 | 否 |

---

## 5. 与审批流程管理的职责边界

### 5.1 审批异常监控职责
- 超时无人审批场景的**监控识别**（监控层）
- 组织变更后在途工单的**监控观察**
- 监控提示与预警生成
- **不直接改变工单结果**

### 5.2 审批流程管理职责
- 审批节点配置与维护
- 审批人分配与推送
- 超时识别与触发（识别层）
- 审批动作的记录与流转

### 5.3 边界总结
- 监控层 = "看" + "提示"
- 流程管理层 = "配" + "推" + "记"
- 工单系统层 = "变" + "承"
