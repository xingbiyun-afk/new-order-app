# 产品申请工单 / 产品申请预算核销 — 产品说明

> 历史说明：本目录早期命名为 `work-order-budget`，用于承接"预算申请工单"原型规划。
> 根据当前最新业务口径，后续统一按"产品申请工单 / 产品申请预算核销"理解。

## 背景

在现有"工单处理中心"基础上，新增"产品申请工单"入口。产品申请工单主要用于预算核销。

## 已实现功能（Vue 参考实现）

### 发起工单页
- 物流工单（已有）
- 客服工单（已有）
- **产品申请工单**（P0+P1 完整实现）
  - 申请信息区：预算未选时压缩，选定后展开
  - 产品申请订单明细：按专卖店拆分，支持添加/折叠/展开
  - 产品选择：搜索 + 卡片列表 + 数量校验
  - 附件上传区：网格布局 + PDF 合并提示
  - **批量导入**：文本粘贴 / Excel/CSV 文件 / 模板下载（CR-20260701-002）
  - 提交校验：金额不超预算可用金额
  - **发起页体验优化**（CR-20260702-002）：无草稿规则、预算/申请信息区压缩展开、滚动预算摘要条、收起态小计、产品术语统一、附件必传感知、提交禁用态、金额预算关系提示、数量提示优化、操作层级区分

### 预算选择页
- **卡片信息层级优化**（CR-20260702-001）：预算标签机制、排序规则、搜索过滤、选中态/不可选态/空态
- 异常状态提示

### 专卖店搜索选择页
- 搜索 + 列表选择 + 回填
- **按 App 现有页面 1:1 回归**（CR-20260703-002）：搜索栏（门店信息标签+搜索框+取消按钮）+ 搜索历史记录 + 行式列表

### 产品搜索选择页
- 搜索 + 卡片列表 + 回填（P1-01）

### 详情页
- 基础信息 / 发起信息 / 审批流 / 订单结果 / 审批操作

### 我的列表页
- Tab：待办 / 已办 / 抄送我的 / 已发起
- 卡片标题 + 三行正文 + 时间右下角 + 单行截断

## 页面路由

## 展示字段

原型中使用以下展示字段（不定义正式枚举）：
- `displayStatus` — 展示状态文本
- `approvalStatusText` — 审批状态文本
- `flowStatusText` — 流程状态文本

## 样式要求

产品申请工单除表单内容、预算核销信息、审批/流转规则、订单生成结果展示不同外，展示风格和流程沿用现有工单样式。
参考 `docs/technical/mobile-ui-baseline.md` 了解完整 UI 规范。
