### 2. CR-20260706-002：审批流、订单结果展示优化与白功能项收口

**分支**：`feature/CR-20260706-002-approval-result-display`

#### 页面改动

- **审批流区节点字段化**：`ApprovalNode` 新增 `nodeName?: string`，模板优先展示 `nodeName`，兜底"审批节点 ${i}"
- **节点类型视觉区分**：新增 `getNodeClass()` 按节点类型和结果返回 CSS 类名（`node-start`/`node-passed`/`node-current`/`node-rejected`）
- **驳回意见紧贴节点**：已驳回场景下，驳回节点的 `.node-remark` 加左侧红色边框，视觉上紧贴驳回节点
- **预占库存订单只展示编号**：`functionOrderNos` (string[]) 数组展示，字号 12px、颜色 `#999`，降为辅助字段；最终审批通过节点不展示预占库存订单
- **失败原因去重**：`failReason` (string) → `failReasons` (string[])，`dedupFailReasons()` 工具函数去重后多个 tip-banner 展示
- **多次失败重试历史**：新增 `OrderAttempt` 接口 + `retryHistory` 字段，折叠收起 + 最终态展示，"订单创建历史 N 次"可点击展开时间线

#### Mock 场景补强

| `completed-retry` | 订单多次失败重试（3 次重试最终成功 + 2 次重试业务放弃） |

#### 白功能项结论

- **`uuid` moderate 风险来源**：`exceljs@4.4.0` 的传递依赖，非当前业务直接依赖
- **不采用 `npm audit fix --force`**：会破坏现有 Excel 导入能力
- **影响范围**：仅 `parseFileImport()` 中的 Excel 解析能力
- **后续建议路径**：关注 `exceljs` 上游修复；评估 `uuid` 升级可行性；考虑 SheetJS 社区版 / PapaParse 纯 CSV 方案
- **本轮结论**：纳入白功能项，形成风险来源、影响范围与后续建议结论，明确本轮不强制修复
