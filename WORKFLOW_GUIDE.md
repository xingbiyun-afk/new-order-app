# WORKFLOW_GUIDE.md — 工作流程指南

## 开发流程

### 1. 需求理解
- 阅读产品文档（`docs/product/`）。
- 确认技术方案（`docs/technical/`）。
- 明确本期范围和不纳入范围的功能。

### 2. 分支创建
- 从 `main` 创建新分支。
- 分支名遵循 `BRANCH_GUIDE.md` 规范。

### 3. 开发实施
- 按 feature 目录组织代码。
- 使用 shadcn/ui 组件库。
- 遵循 `CLAUDE.md` 中的状态字段和样式约定。
- 保持移动端优先的设计理念。

### 4. 构建验证
- 开发完成后执行 `npm run build`。
- 确保构建无错误。

### 5. 提交规范
- 提交信息格式：`{type}: {description}`。
- 示例：`feat: add budget work order entry page`。
