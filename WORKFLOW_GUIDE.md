# WORKFLOW_GUIDE.md — 工作流程指南

## 当前项目定位

`new-order-app` 当前不是正式生产交付仓，而是：

1. 产品申请工单 / 产品申请预算核销需求文档沉淀仓；
2. 基于 Vue 的移动端样式、交互、结构参考实现仓。

当前阶段允许直接在本仓库中开展 Vue 参考实现，用于需求验证、交互确认、页面走查和后续正式开发承接。

## 当前正式业务口径

- 当前正式对象为：`产品申请工单` / `产品申请预算核销`
- 不是：`预算申请工单`
- 不恢复以下旧概念：
  - 预算责任人管理
  - 主责 / 备责
  - 责任人异常影响预算核销
  - 待调整预算状态
- 发起人与审批人来源以：`产品申请审批流程管理` 为准
- App 当前参考实现与后续正式开发方向：`Vue`

## 开发启动机制

当前阶段采用“首轮开发任务书 + 后续变更清单”模式：

1. 首轮开发由 `docs/product/work-order-budget/产品申请工单AI开发任务说明.md` 启动；
2. AI 先按该任务书完成第一轮 Vue 参考实现开发；
3. 首轮开发完成后进入验收；
4. 在该任务书基础上新增的需求、修订、补充、纠偏，不重写新的主任务书，而通过 `docs/change-logs/` 下的变更清单持续记录。

- 遗留问题
- 后续建议

目录建议：
- `docs/development-summary/`

### 4. 项目日报
当天结束后补充项目日报，记录：

- 今日工作概览
- 已完成工作

目录建议：
- `docs/daily-reports/`

## 标准开发流程

### 1. 需求理解
- 阅读产品文档（`docs/product/`）
- 阅读技术文档（`docs/technical/`）
- 阅读 AI 协作基线与工作流文档
- 明确本期范围和不纳入范围的功能

### 2. 任务启动
- 确认当前是否已有正式开发任务说明
- 若是首轮开发，先按开发任务说明执行
- 若是后续调整，新增变更清单后再执行

### 3. 分支创建
- 从 `main` 创建新分支
- 分支名遵循 `BRANCH_GUIDE.md` 规范

### 4. 开发实施
- 按 feature 目录组织代码
- 当前 App 参考实现以 Vue 为准
- 优先复用现有工单移动端交互风格
- 保持移动端优先的设计理念
- 不擅自扩展未确认业务规则

### 5. 构建验证
- 开发完成后执行 `npm run build`
- 确保构建无错误

### 6. 提交规范
- 提交信息格式：`{type}: {description}`
- 示例：`feat: add product application create page`

### 7. 开发总结
- 每轮开发完成后，在 `docs/development-summary/` 下新增开发总结文档

### 8. 每日收尾
- 当天结束后，在 `docs/daily-reports/` 下新增或更新项目日报
